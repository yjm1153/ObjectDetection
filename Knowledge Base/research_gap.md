# Research Gap Analysis
> 每读一篇论文必须更新。不要复制论文，必须提出自己的观点。

每篇论文回答5个问题：
1. **别人没解决什么？** 为什么？
2. **还有哪些瓶颈？** 技术/数据/评估层面
3. **为什么没人做？** 技术壁垒？缺乏数据？社区忽视？
4. **如果我是作者，下一篇会做什么？** 自然延伸方向
5. **提出≥3个新Idea**（记录到 Ideas/candidate.md）

## Gap 记录

### 来自 SEEN-DA(CVPR 2025)
1. **别人没解决什么?** 语义信息(VLM 文本嵌入)只被用于检测头/域对齐,从未进入实时检测器(YOLO 系)的特征提取与融合阶段。SEEN-DA 也只做了 Faster R-CNN。为什么:DAOD 社区默认两阶段基座;YOLO 社区少有人碰 VLM。
2. **瓶颈**:语义熵对小目标天然偏高(特征弱→分布均匀→高熵),会被误判为冗余;CLIP 对遥感/无人机视角图像的文本对齐质量未知(训练分布外)。
3. **为什么没人做?** VLM 推理开销与 YOLO 实时性表面冲突——但文本嵌入可离线缓存,熵图计算只是一次矩阵乘+softmax,实际代价很低。这是被"直觉"挡住的方向。
4. **如果我是作者,下一篇做什么?** 把语义熵做成通用即插即用模块(免 VLM 版:类别原型熵),或做熵引导的稀疏推理。
5. 新 Idea → 见 Ideas/candidate.md #5–#8。

### 来自 SEMA-YOLO(Remote Sens. 2025)
1. **别人没解决什么?** P2 头的算力代价问题:所有"加 P2 头"的工作(RSI-YOLO 等)都接受 GFLOPs 翻倍,没人做 P2 层的稀疏化/条件计算。截短 backbone 只被 SEMA 顺手做了,没有系统研究"小目标场景的最优深度-宽度-分辨率再分配"。
2. **瓶颈**:评估层面——AI-TOD 被随意重划分子集,遥感小目标论文之间结果不可比;VisDrone/DOTA 上缺统一协议的 P2 改进对照。
3. **为什么没人做?** 结构搜索算力门槛+社区满足于"模块拼装涨点"的发表模式;稀疏计算在稠密预测头上工程实现难。
4. **如果我是作者,下一篇做什么?** RFA-ASFF 协同机制分析(为什么 RFA 单用掉点);SLE 的适用边界(大小目标混合时何时失效)。
5. 新 Idea → 见 Ideas/candidate.md #9–#12。

### 来自 SFIDM(Remote Sens. 2025)
1. **别人没解决什么?** 频域信息只在图像级粗糙使用(整图 DFT 切两半),没人做特征级/多尺度频域分解;三路特征提取的冗余无人优化;频率切分阈值固定,无自适应机制。为什么:频域方法在检测社区长期边缘化,做的人少,还停在"能用就行"阶段。
2. **瓶颈**:技术层面——图像级 DFT 丢失空间-频率局部性(全图一个阈值);评估层面——FSOD 用 VOC2007 11-point 协议,与主流 COCO 协议不可比,遥感 FSOD 结果普遍虚高。
3. **为什么没人做?** 频域与空域特征的语义鸿沟(频域特征难以直接解释);小波/DFT 在 GPU 上的高效实现门槛;社区注意力集中在注意力机制和 VLM。
4. **如果我是作者,下一篇做什么?** 特征级频域分解(FcaNet 路线)替代图像级;高频能量图直接作为空间显著性先验,而不是再训练一条 CNN 去"学"高频特征。
5. 新 Idea → 见 Ideas/candidate.md #11–#12。

### 交叉 Gap(两篇共同启发,价值最高)
- **语义熵 × 小目标结构优化**:SEMA 的 P2 头带来 4 倍面积的浅层特征图,其中大量是背景;SEEN-DA 的语义熵恰好是"免训练的背景识别器"。**用(免 VLM 的)语义/原型熵在 P2–P3 特征图上做稀疏化或融合加权,同时解决 SEMA 的算力问题和小目标的背景干扰问题**——两篇论文各自的最大短板互为解药,且无人做过。
- **SFIDM 补充(2026-07-15)**:高频能量图是第二种免训练前景判据(只需一次 DFT,零可学习参数、零 VLM 依赖)——为交叉 Gap 提供了"语义熵失效时的备胎判据",也可与语义熵互补(语义确认"是什么",高频确认"在哪里")。

### Idea #5 查新结果(2026-07-15,WebSearch)
**结论:novelty 成立**。「VLM 语义熵 × YOLO P2 空间特征稀疏化(条件计算省 FLOPs)」组合未被覆盖。最近邻工作及划界:
| 最近邻 | 做了什么 | 与 #5 的本质区别 |
|--------|---------|----------------|
| ViCrop-Det(最接近) | DETR 解码器注意力熵→高熵区域裁剪重检测(training-free) | 推理级**图像裁剪路由**,不进网络内部;仅 DETR 系;熵源是解码器注意力而非 VLM 语义 |
| SViT(WACV'24)/ Token Cropr(CVPR'25,✅已精读) | ViT backbone 的 token pruning(检测任务);Token Cropr:单query router+辅助头(训练后丢弃)+LLF 末端复活,COCO 97%剪枝 63.0 AP_box | 四重区别:①ViT token vs CNN 多尺度特征图;②可学习 router(任务监督)vs 免训练熵/高频先验;③backbone 内 vs neck/head P2 分支;④1536²非实时未验证小目标 vs VisDrone 实时小目标;其 LLF 机制可借鉴作 #5 误剪兜底 |
| BiSD-YOLO(SDGE) | 稀疏动态门控做背景抑制(航拍小目标) | 特征增强模块,不做条件计算、不省 FLOPs;非熵引导 |
| PEEK / WCE 熵剪枝 | 熵用于静态结构化剪枝(剪通道/层) | #5 是空间维、样本自适应的动态稀疏,正交 |
| **YOLOE-LRPC(2025,新增)** | 专用嵌入语义分数滤 anchor(`o·P_sᵀ>δ` 滤 80%,无损提速 1.7×) | 过滤发生在**头部输出级**(anchor 分类前),目的是省词表检索;backbone/neck 稠密计算一点没省。#5 在**特征级**(P2 计算前)稀疏化,省的是稠密卷积 FLOPs——层级、对象、目的均不同 |

- 佐证:ViCrop-Det 的熵引导使 VisDrone/DOTA APS +3.8~4.8 → 熵-小目标关联可行性有旁证。
- 佐证(YOLOE,2025-新增):LRPC 用 δ=0.001 的极低阈值即无损滤掉 80% anchor → **背景与前景的语义分数分布高度可分**,间接支持 #5 的熵判据假设;同时证明"逐点语义分数过滤"在 YOLO 上工程可行且无损。
- 写论文时 Related Work 必须引用以上五线并划界;SViT 的「token 保留/再激活」机制(防误剪)值得借鉴到特征稀疏化(误剪小目标的兜底)。

### 来自 YOLO-World(CVPR 2024)
1. **别人没解决什么?** ①文本/语义只注入 PAN 的 P3–P5,**backbone 与 P2 浅层完全无语义引导**;②开集 YOLO 无 P2 层,小目标(LVIS 未报 AP_s)是盲区;③I-Pooling 把全图压成 3×3,图像→文本方向融合极粗糙;④跨模态融合在小词表下冗余(COCO fine-tune 被移除)却无自适应开关机制。
2. **瓶颈**:P2 下探 = 4 倍特征面积 × 跨模态计算,算力直接爆炸——**没有稀疏化就下不去,这正是 #5 的位置**。
3. **为什么没人做?** 开集社区在拼大词表 zero-shot 榜(LVIS/ODinW),小目标不在其评估叙事里;实时约束下没人敢加 P2。
4. **如果我是作者,下一篇做什么?** ①P2 + 熵引导稀疏化(=#5);②词表大小自适应的融合开关;③熵/不确定性感知的 I-Pooling 采样。
5. **结构层面确认 #5 的 Gap**:YOLO-World(最强开集 YOLO 基座)与「语义熵 × P2 稀疏化」完全正交——#5 可直接以其为基座,Related Work 划界干净。

### 来自 TinyFormer (arXiv 2026.05) —— ⚠️ 架构演进信号
1. **别人没解决什么?** YOLO 和 DETR 在微小目标上的失败机制被首次系统分析(空间退化+token 坍缩),但 PBM 的"浅层捷径"和显式 P2 头的关系未被讨论——**两者是功能重叠还是互补?** 无人回答。
2. **瓶颈**:PBM 用跨层捷径替代显式 P2 头——保留了空间细节但**所有位置都保留**(无选择性);#5 的 P2+熵稀疏化是选择性保留(只在前景区)。这是"稠密传递 vs 稀疏传递"的路线分歧。
3. **为什么没人做?** YOLO-DETR 混合架构刚出现,社区还在探索 backbone+neck 的最优组合,未深入到稀疏化。
4. **如果我是作者,下一篇做什么?** PBM + 显式 P2 头的叠加收益分析;PBM 的选择性版本(只传高信息量捷径——即 #5 的思路)。
5. **新 Gap**:PBM 在纯 CNN backbone(非 ViT)上的有效性未知——CNN 的浅层特征与 ViT 的 token 在信息密度上不同,PBM 可能需要在 CNN 上重新设计。

### 来自 DERNet (arXiv 2026.06) —— #11 最强竞争者
1. **别人没解决什么?** 频域处理只在特征增强维度,从未用于**计算资源分配**。DERNet 的 WDG/LGE/FDHead 全部做"频域→更好的特征",不做"频域→决定要不要算"。
2. **瓶颈**:小波/Log-Gabor 在 GPU 上的 wall-clock time 可能显著高于理论 FLOPs(小波变换是 memory-bound 操作)。P2 分辨率是 P5 的 16 倍——DERNet 的频域处理在 P2 上的实际速度是未知数。
3. **为什么没人做?** 频域+检测是 2025–2026 才热的方向,目前大家都在探索"怎么用频域提升特征质量",还没人想到"用频域省算力"。
4. **如果我是作者,下一篇做什么?** 频域引导的自适应计算分配(高频区域→精细处理,低频区域→粗处理)——但 DERNet 自己还没走到这步,#11 可以抢先。
5. **"1/6 参数超越 YOLOv11" 的背面**:参数少≠算力少——小波变换工程效率是关键限制。

### 来自 SET (CVPR 2025) —— ⚠️ 对 #11 的根本性质疑
1. **别人没解决什么?** "移除高频→帮助小目标"的反直觉发现被提出,但**这个结论在 P2 层是否成立不知道**——P2 未经多次下采样,高频噪声累积少,结论可能反转。
2. **瓶颈**:HBS 依赖精确 GT box mask——小目标标注噪声(VisDrone 常见)下 HBS 可能误平滑前景边缘。
3. **为什么没人做?** 频域分析在检测中刚起步,大家都从 FPN 中间层开始分析,P2(最高分辨率)的计算开销让频域分析望而却步。
4. **如果我是作者,下一篇做什么?** 自适应频谱策略:对极小目标抑制高频噪声(P2),对大目标保留高频细节(P5)——尺度特异的频域处理。这正是 #11 可以做的(高频判据+尺度自适应)。
5. **#11 的修正方案**:SET 证明"绝对高频能量"作判据有问题→应改用**"高频能量局部异常度"**(小目标高频在空间上点状孤立,背景高频规则分布)。

### 来自 D3Q (IEEE JSTARS 2025) —— DETR 线小目标能力定位
1. **别人没解决什么?** DETR 的动态查询解决了"查询数量 vs 目标密度"的矛盾,但**小目标特征质量本身**(token 坍缩、空间退化)没改善——D3Q 是"给更多查询去搜索",而非"让每个查询看得更清"。
2. **瓶颈**:动态查询依赖密度估计(需 GT 监督)——无监督/自监督密度估计(用熵/频域先验)可替代但无人尝试。
3. **为什么没人做?** DETR 社区在解决收敛速度/训练效率(DINO/DN-DETR/DAB-DETR),小目标专项优化留给了 CNN 系。
4. **如果我是作者,下一篇做什么?** D3Q 的密度估计用频域/熵先验替代 GT 监督→零标注开销的查询数量自适应。
5. **对"坚持 YOLO"决策的影响**:D3Q(49M/543G)vs YOLO11n(2.6M/6.3G)——DETR 在 VisDrone 上无压倒性优势,轻量/实时场景 YOLO 不可替代。**决策不变。**

### 来自 SFDNet (ECCV 2026) —— 频谱解耦分治法
1. **别人没解决什么?** 三频解耦(低/中/高)比单一高频判据精细,但 SFDNet 只做特征增强——**频谱信息用于条件计算/稀疏化的潜力完全未探索**。
2. **瓶颈**:三路处理的计算开销(低/中/高三条并行通路)可能抵消频谱解耦的优势;CPD 原型蒸馏在类别数大的场景(COCO 80 类)效果未知。
3. **为什么没人做?** 频域分治 + 原型蒸馏是新的技术组合,刚被提出,社区还在消化。
4. **如果我是作者,下一篇做什么?** ASD 的频谱权重用熵/频域先验替代学习(免训练→零参数→更轻量),原型蒸馏用 CLIP 文本原型替代视觉原型(零 shot)。
5. **#11 的升级路线**:从"高频 vs 低频"的二元判据→多频谱能量分布作向量判据(每个位置的频谱签名)。

### ⚠️ 频域小目标浪潮: 3篇→4篇 (2026-07-16 更新)

**现象**:至少 4 篇独立工作(SET CVPR 2025 + FMC-DETR arXiv 2025.09 + DERNet arXiv 2026.06 + SFDNet ECCV 2026)在同一时间窗口内瞄准"频域+小目标检测",且彼此无相互引用(投稿时间线重叠)。**频域小目标检测正处于早期快速上升期。**

**四篇的共同点**:
- 都承认频域信息对小目标定位至关重要
- 都用频域做**特征增强**(更好的特征→更准的检测)
- 都有即插即用设计

**四篇都没做的事(这正是 #11 的位置)**:
- 用频域信息做**条件计算/稀疏化**(频域→判定重要性→跳过冗余计算)
- 在 **P2 层**专门做频域分析(P2 分辨率太高,四篇都回避了)
- 将频域判据与**语义判据**(CLIP 熵)对比/融合

**#11 的差异化生命线**:Related Work 引这四篇,承认频域对小目标有效,然后说"但现有工作只在增强维度使用频域,其作为**计算资源分配依据**的潜力未被探索——本文首次提出频域引导的特征级条件计算,在保持小目标精度的同时大幅降低 P2 头的算力开销。"

**叙事策略建议**:
- #11 的标题/摘要必须出现 "frequency-guided conditional computation" 或 "spectral sparsification" 以与"frequency enhancement"路线区隔
- Related Work 需显式划界:"不同于 SET/FMC-DETR/DERNet/SFDNet 利用频谱改善特征质量,本文利用频谱改善计算效率"

### 来自 FMC-DETR (arXiv 2025.09) —— 振幅-相位解耦 × 条件计算

1. **别人没解决什么?** ①MDFC的振幅-相位解耦极优雅(振幅="改多少",相位="在哪"),但全图统一调制无空间选择性;②WeKat的小波递归分解(HSG-WAVE)在浅层扩大ERF+抑制噪声,但小波特征未被用于判据;③DETR侧也发现D2(浅层高分辨率)关键,验证了P2/D2的跨架构价值。
2. **瓶颈**: DETR-based,FPS仅17.8;小波GPU墙钟效率未消融;MDFC三路并行+CPF四分支的计算开销。
3. **为什么没人做?** 频域+检测社区正探索"怎么用频域提特征质量",未想到"用频域省算力"。
4. **如果我是作者,下一篇做什么?** ①振幅引导的空间选择性FFT(高频区域→精细频域分析)→这正是#11;②YOLO版轻量MDFC→YOLO head。
5. **新交叉 Gap: 振幅调制 × 语义熵 × 高频判据**——三种门控范式(频域振幅/语义熵/高频能量)功能同构但信息来源互补,可AND/OR融合。

### ⚠️ 对#11叙事的影响: 4篇Related Work证据链完整

Related Work更加充实——"已有4篇独立工作(SET/FMC-DETR/DERNet/SFDNet)共同证明频域信息对小目标检测至关重要,但它们仅在特征增强维度利用频域。本文首次将频域信息用作计算资源分配的依据..."

### 来自 ViCrop-Det (arXiv 2026.04) —— 注意力熵 × 语义熵 交叉路口

1. **别人没解决什么?** ①SAE是DETR专属(YOLO无cross-attn decoder),注意力熵范式在单阶段检测器上完全空白;②零注意力盲区(模型完全不关注的目标)是SAE的硬伤,VLM语义先验可覆盖此类盲区;③SAE仅用于推理时裁剪路由,从未被建模为模型内部的特征门控信号。
2. **瓶颈**: YOLO没有cross-attention→SAE无法直接移植;但YOLO可以用**语义熵**(来自VLM)或**特征能量熵**(免VLM)作为替代判据。
3. **为什么没人做?** YOLO社区未接触注意力熵概念(ViCrop-Det也是DETR专属);VLM社区不做特征级稀疏化。
4. **如果我是作者,下一篇做什么?** ①将SAE替换为语义熵(CLIP嵌入熵)→覆盖零注意力盲区→提升全面性;②将裁剪路由替换为特征级门控→嵌入模型内部→减少推理开销。
5. **新 Gap: 注意力熵×语义熵双源门控的交叉点**——两者功能同构但信息来源互补(模型内生 vs 外部知识),AND/OR融合即可超越单一判据。**这直接强化Idea#5的叙事**,且ViCrop-Det可作正交对比引用。

### 来自 D³R-DETR (arXiv 2026.01) —— Gabor 频域核 × 密度稀疏化

1. **别人没解决什么?** ①Gabor核在频域核选型中的优越性(Fourier/Haar/Gabor)已被验证但仅限于DETR密度估计,未用于条件计算;②密度热力图引导的稀疏化(MWAS)在DETR验证有效但YOLO无对应实现;③频域分支(FPU)与空间分支(DilatedSPU)的融合方式粗暴(直接concat),无自适应门控。
2. **瓶颈**: Gabor核参数(kernel size/scale/angle)需手工选择,无自适应机制;DETR专属;仅AI-TOD-v2验证。
3. **为什么没人做?** 频域检测起步晚,Gabor核在检测中的系统研究几乎空白。
4. **如果我是作者,下一篇做什么?** ①Gabor→自适应参数选择(学习式);②密度稀疏化→YOLO P2层移植;③双域融合→振幅-相位门控替代concat。
5. **对#11/#13的指导**: Gabor核作为#13振幅-相位解耦的备选频域工具;D³R-DETR证实"密度引导特征稀疏化"有效→#11可借鉴其稀疏化范式但用频域判据替代密度判据。

### 来自 DM-EFS (ICCV 2025) —— P2 浅层特征价值验证

1. **别人没解决什么?** ①浅层高分辨率特征的价值被EFS验证,但DM-EFS的选择依据(图像级尺寸codebook)过于粗糙→按token级语义/频域动态选择是下一步;②Size-Codebook是离散查表(3档),连续软门控未被探索;③仅按尺寸忽视语义内容→同样尺寸目标可能有不同特征需求。

### 来自 O² (TGRS 2026) —— DETR 实时旋转检测

### 来自 MOCHA (arXiv 2025/2026) —— VLM→YOLO 蒸馏技术基线

1. **别人没解决什么?** ①MOCHA是VLM蒸馏到YOLO的成功先例,但仅做**均匀**特征对齐→未考虑不同区域的蒸馏难度差异;②小目标区域VLM特征更丰富(相比CNN)→应获得更高蒸馏权重→MOCHA未做;③Translation Module仅用于训练,其"跨模态桥接"能力可否反向利用(VLM→检测器之外的交互)?

### 来自 CLIP-Bias (arXiv 2026.07) —— Idea#5 的"定时炸弹"与应对

1. **别人没解决什么?** CLIP特征在检测中的结构性偏差被首次诊断,但修正方案(温度缩放)不彻底→真正的解决方案需要**重新设计特征使用方式而非后修正**。
2. **瓶颈**: 偏差仅能部分纠正(Recall vs Precision权衡);遥感场景未验证(俯视视角可能偏差更严重);修正后的小目标R@10仍远低于大目标。
3. **为什么没人做(彻底解决)?** VLM的预训练方式(image-level对比学习)决定了其区域级偏差是结构性的→彻底解决需要改预训练或重新设计downstream使用方式。
4. **如果我是作者,下一篇做什么?** ①规范化语义熵(用attention entropy作为归一化reference);②遥感场景的CLIP偏差诊断(preliminary study);③双源门控(语义熵+高频能量)→任一源有偏差时另一源补偿。
5. **对Idea#5的双重影响**:
   - ⚠️ **负面影响**: CLIP特征天然歧视小目标→语义熵门控可能不可靠→Idea#5的风险显著上升
   - ✅ **正面影响**: ①CLIP偏差+小目标检测的结合点**未见于任何已查论文**→Idea#5的差异化叙事增强;②CLIP偏差恰好为Idea#5的"语义熵归一化+尺度校正"机制提供了充分的motivation→问题越严重,解决方案越有价值;③双源门控(#15)天然规避→提升优先级

### 来自 DINOv2 (ICCV 2023) ⚠️ pre-2025 路径三 P0
1. **别人没解决什么?** CLIP/VLM 文本嵌入对小目标的系统性偏差已被证实(CLIP-Bias: r=0.579)——但没人提供第三种"中立"判据。DINOv2 的自监督 patch features 自动涌现 objectness(PCA 前景分离,零文本监督),恰好规避 CLIP 的尺度偏差,但至今无人将其用作检测器内部的条件计算判据。
2. **瓶颈**: ViT 离线特征提取的 wall-clock overhead 未知;PCA 前景分离在密集多人/多物场景下可能误判;DINOv2 自身对极小目标(<8×8)的 patch 质量未验证。

4. **如果我是作者,下一篇做什么?** 三判据对照实验(CLIP熵 vs DINOv2-PCA vs FFT频谱)→ 锁定小目标最佳判据 → 融合或替换。

### 来自 Focal Loss (ICCV 2017) ⚠️ pre-2025 路径三 P0
1. **别人没解决什么?** 条件计算/动态网络社区全部用"节省算力"作为动机(engineering-driven)，无人将 Focal Loss 的"难例聚焦"数学框架从**损失空间**推广到**计算空间**。Token Cropr、ViCrop-Det、MGS 都是硬剪枝(类比 OHEM 硬采样)，无人做软加权计算分配(类比 Focal Loss 软降权)。
2. **瓶颈**: 类比不等于等价——损失空间的重心偏向与计算空间的资源分配是不同的数学问题；需要 P2 层真实 token 熵分布数据验证类比是否成立。
3. **为什么没人做?** Focal Loss 已被社区默认为"分类 loss 专用工具"；17 年前的论文,认知惯性使社区"遗忘"其泛化原理("any loss with similar properties works")。
4. **如果我是作者,下一篇做什么?** 提出 Focal Computation 理论框架：FC(i)=α(pos_i)·(1-E(H_i))^γ·FLOPs_base → 将所有 P2 稀疏化方法统一到同一数学语言下。
5. → Idea #20 (Focal Computation 理论框架)。

### 来自 FcaNet (ICCV 2021) ⚠️ pre-2025 路径三 P1
1. **别人没解决什么?** 频域检测浪潮 4 篇全部在**空间维度**做频域变换,无人将频域分析引入**通道维度**做条件计算。FcaNet 证明不同通道对不同 2D DCT 频率分量有不同敏感性→可直接推广到稀疏化(某些通道只对特定频率敏感→可选择性关闭)。

3. **为什么没人做?** 通道注意力社区认为"已无新意";频域检测刚兴起,交叉到通道维度需要时间。
4. **如果我是作者,下一篇做什么?** FcaNet 频域通道选择 × #11 频域空间选择 → 双维稀疏化(空间×通道联合门控)。

### 来自 SViT (WACV 2024) ⚠️ pre-2025 路径三补充
1. **别人没解决什么?** SViT仅用于ViT backbone token pruning(分类→检测的迁移)。其四大原则——特别是Token再激活和动态剪枝率——从未被应用于**YOLO P2检测头的空间token稀疏化**。Token Cropr的LLF是被剪token在末层前原位置插入;SViT的再激活是逐层独立选择——两者互补但都未被引入YOLO。
2. **瓶颈**: Gumbel Softmax选择器需训练(不同于#5的免训练语义熵);ViT token≠CNN空间特征图token(语义不同,空间对齐不同)。
3. **为什么没人做?** Token pruning社区聚焦ViT+分类;YOLO社区不用token pruning概念。
4. **如果我是作者,下一篇做什么?** 混合门控:语义熵初筛(免训练)+ Gumbel Softmax再激活(可学习)→取两者之长。

### 来自 DFIR-DETR (arXiv 2026.05) —— 频域最小二乘×动态稀疏
1. **别人没解决什么?** ①FIRC3的频域最小二乘框架仅用于特征聚合,未被推广到门控/条件计算;②DKSA的K预测门控直接验证了Focal Computation范式但未被识别为该范式的实例;③ANUP振幅归一化解决了上采样特征膨胀,但YOLO P2上采样(最近邻/双线性)无类似修正。

3. **为什么没人做?** DETR和YOLO社区的方法论隔离——DFIR-DETR的贡献被理解为"DETR改进",而非"可跨架构迁移的频域优化范式"。
4. **如果我是作者,下一篇做什么?** ①FIRC3→P2频域门控的数学框架(将频域门控表述为约束最小二乘);②DKSA→P2 Focal Computation的第一个工程实例;③ANUP→YOLO P2上采样振幅修正。
5. **新 Gap: 频域最小二乘门控框架**——将#11的"高频能量判据"升级为"频域约束最小二乘门控",数学上更严格,且与DFIR-DETR的FIRC3形成引用链。

### 来自 HashEye (SciRep 2026) —— 输入级空间剪枝(2026-07-17 轻量化首轮)
1. **别人没解决什么?** ①LSH 剪枝判据是纯低层纹理统计,**复杂城市地形失效**(论文自认)——语义级判据在城市场景的空间剪枝仍空白(=#5 主场);②剪枝发生在输入级,特征级空间稀疏化依旧无人做;③逐帧独立,时序冗余未开发(=#29 的 Gap 被该文 future work 背书)。
2. **瓶颈**: 判据无语义(纹理重复≠无目标);目标须 <40×40 patch;mosaic 破坏全局上下文→高 FP。
3. **为什么没人做?** 端侧部署社区从系统角度(哈希/内存拷贝)切入,不碰特征语义;检测社区不做输入级预处理。
4. **如果我是作者,下一篇做什么?** 可学习 LSH(投影矩阵适应环境纹理)/时序缓存哈希图/自适应 patch 尺寸。
5. **工程结论(对 #5 v3.0 直接有效)**: masked conv 不省 GPU 时间 + 稀疏 kernel warp divergence → 唯一已验证路线是**稠密重排**;且预处理占 53% 延迟仍净赚 5.25×,证明"判据计算开销大也值得"——#5 的语义熵计算开销预算可放宽。

1. **别人没解决什么?** ①packing(特征级稠密重排)只用于 ViT token,CNN 特征图空间稀疏化的 packing 版本无人做;②GT 监督门控只在 Swin 深层验证,**浅层被作者主动放弃**("过早剪=信息损失")——高分辨率浅层的可剪性是公开难题;③选择标签来自 GT 框栅格化,免监督判据(熵/频域)未探索。

### 来自 DQ-DETR (ECCV 2024) ⚠️ pre-2025 高质量
