## Gap 记录

### 来自 SEEN-DA(CVPR 2025)
1. **别人没解决什么?** 语义信息(VLM 文本嵌入)只被用于检测头/域对齐,从未进入实时检测器(YOLO 系)的特征提取与融合阶段。SEEN-DA 也只做了 Faster R-CNN。为什么:DAOD 社区默认两阶段基座;YOLO 社区少有人碰 VLM。
2. **瓶颈**:语义熵对小目标天然偏高(特征弱→分布均匀→高熵),会被误判为冗余;CLIP 对遥感/无人机视角图像的文本对齐质量未知(训练分布外)。
3. **为什么没人做?** VLM 推理开销与 YOLO 实时性表面冲突——但文本嵌入可离线缓存,熵图计算只是一次矩阵乘+softmax,实际代价很低。这是被"直觉"挡住的方向。
4. **如果我是作者,下一篇做什么?** 把语义熵做成通用即插即用模块(免 VLM 版:类别原型熵),或做熵引导的稀疏推理。
5. 新 Idea → 见 Ideas/candidate.md #5–#8。

### 来自 SEMA-YOLO(Remote Sens. 2025)
1. **别人没解决什么?** P2 头的算力代价问题:所有"加 P2 头"的工作(RSI-YOLO 等)都接受 GFLOPs 翻倍,没人做 P2 层的稀疏化/条件计算。截短 backbone 只被 SEMA 顺手做了,没有系统研究"小目标场景的最优深度-宽度-分辨率再分配"。
2. **瓶颈**:评估层面——AI-TOD 被随意重划分子集,遥感小目标论文之间结果不可比;VisDrone/DOTA 上缺统一协议的 P2 改进对照。
3. **为什么没人做?** 结构搜索算力门槛+社区满足于"模块拼装涨点"的发表模式;稀疏计算在稠密预测头上工程实现难。
4. **如果我是作者,下一篇做什么?** RFA-ASFF 协同机制分析(为什么 RFA 单用掉点);SLE 的适用边界(大小目标混合时何时失效)。
5. 新 Idea → 见 Ideas/candidate.md #9–#12。

### 来自 SFIDM(Remote Sens. 2025)
1. **别人没解决什么?** 频域信息只在图像级粗糙使用(整图 DFT 切两半),没人做特征级/多尺度频域分解;三路特征提取的冗余无人优化;频率切分阈值固定,无自适应机制。为什么:频域方法在检测社区长期边缘化,做的人少,还停在"能用就行"阶段。
2. **瓶颈**:技术层面——图像级 DFT 丢失空间-频率局部性(全图一个阈值);评估层面——FSOD 用 VOC2007 11-point 协议,与主流 COCO 协议不可比,遥感 FSOD 结果普遍虚高。
3. **为什么没人做?** 频域与空域特征的语义鸿沟(频域特征难以直接解释);小波/DFT 在 GPU 上的高效实现门槛;社区注意力集中在注意力机制和 VLM。
4. **如果我是作者,下一篇做什么?** 特征级频域分解(FcaNet 路线)替代图像级;高频能量图直接作为空间显著性先验,而不是再训练一条 CNN 去"学"高频特征。
5. 新 Idea → 见 Ideas/candidate.md #11–#12。

### 交叉 Gap(两篇共同启发,价值最高)
- **语义熵 × 小目标结构优化**:SEMA 的 P2 头带来 4 倍面积的浅层特征图,其中大量是背景;SEEN-DA 的语义熵恰好是"免训练的背景识别器"。**用(免 VLM 的)语义/原型熵在 P2–P3 特征图上做稀疏化或融合加权,同时解决 SEMA 的算力问题和小目标的背景干扰问题**——两篇论文各自的最大短板互为解药,且无人做过。
- **SFIDM 补充(2026-07-15)**:高频能量图是第二种免训练前景判据(只需一次 DFT,零可学习参数、零 VLM 依赖)——为交叉 Gap 提供了"语义熵失效时的备胎判据",也可与语义熵互补(语义确认"是什么",高频确认"在哪里")。

### Idea #5 查新结果(2026-07-15,WebSearch)
**结论:novelty 成立**。「VLM 语义熵 × YOLO P2 空间特征稀疏化(条件计算省 FLOPs)」组合未被覆盖。最近邻工作及划界:
| 最近邻 | 做了什么 | 与 #5 的本质区别 |
|--------|---------|----------------|
| ViCrop-Det(最接近) | DETR 解码器注意力熵→高熵区域裁剪重检测(training-free) | 推理级**图像裁剪路由**,不进网络内部;仅 DETR 系;熵源是解码器注意力而非 VLM 语义 |
| SViT(WACV'24)/ Token Cropr(CVPR'25,✅已精读) | ViT backbone 的 token pruning(检测任务);Token Cropr:单query router+辅助头(训练后丢弃)+LLF 末端复活,COCO 97%剪枝 63.0 AP_box | 四重区别:①ViT token vs CNN 多尺度特征图;②可学习 router(任务监督)vs 免训练熵/高频先验;③backbone 内 vs neck/head P2 分支;④1536²非实时未验证小目标 vs VisDrone 实时小目标;其 LLF 机制可借鉴作 #5 误剪兜底 |
| BiSD-YOLO(SDGE) | 稀疏动态门控做背景抑制(航拍小目标) | 特征增强模块,不做条件计算、不省 FLOPs;非熵引导 |
| PEEK / WCE 熵剪枝 | 熵用于静态结构化剪枝(剪通道/层) | #5 是空间维、样本自适应的动态稀疏,正交 |
| **YOLOE-LRPC(2025,新增)** | 专用嵌入语义分数滤 anchor(`o·P_sᵀ>δ` 滤 80%,无损提速 1.7×) | 过滤发生在**头部输出级**(anchor 分类前),目的是省词表检索;backbone/neck 稠密计算一点没省。#5 在**特征级**(P2 计算前)稀疏化,省的是稠密卷积 FLOPs——层级、对象、目的均不同 |

- 佐证:ViCrop-Det 的熵引导使 VisDrone/DOTA APS +3.8~4.8 → 熵-小目标关联可行性有旁证。
- 佐证(YOLOE,2025-新增):LRPC 用 δ=0.001 的极低阈值即无损滤掉 80% anchor → **背景与前景的语义分数分布高度可分**,间接支持 #5 的熵判据假设;同时证明"逐点语义分数过滤"在 YOLO 上工程可行且无损。
- 写论文时 Related Work 必须引用以上五线并划界;SViT 的「token 保留/再激活」机制(防误剪)值得借鉴到特征稀疏化(误剪小目标的兜底)。

### 来自 YOLO-World(CVPR 2024)
1. **别人没解决什么?** ①文本/语义只注入 PAN 的 P3–P5,**backbone 与 P2 浅层完全无语义引导**;②开集 YOLO 无 P2 层,小目标(LVIS 未报 AP_s)是盲区;③I-Pooling 把全图压成 3×3,图像→文本方向融合极粗糙;④跨模态融合在小词表下冗余(COCO fine-tune 被移除)却无自适应开关机制。
2. **瓶颈**:P2 下探 = 4 倍特征面积 × 跨模态计算,算力直接爆炸——**没有稀疏化就下不去,这正是 #5 的位置**。
3. **为什么没人做?** 开集社区在拼大词表 zero-shot 榜(LVIS/ODinW),小目标不在其评估叙事里;实时约束下没人敢加 P2。
4. **如果我是作者,下一篇做什么?** ①P2 + 熵引导稀疏化(=#5);②词表大小自适应的融合开关;③熵/不确定性感知的 I-Pooling 采样。
5. **结构层面确认 #5 的 Gap**:YOLO-World(最强开集 YOLO 基座)与「语义熵 × P2 稀疏化」完全正交——#5 可直接以其为基座,Related Work 划界干净。

### 来自 TinyFormer (arXiv 2026.05) —— ⚠️ 架构演进信号
1. **别人没解决什么?** YOLO 和 DETR 在微小目标上的失败机制被首次系统分析(空间退化+token 坍缩),但 PBM 的"浅层捷径"和显式 P2 头的关系未被讨论——**两者是功能重叠还是互补?** 无人回答。
2. **瓶颈**:PBM 用跨层捷径替代显式 P2 头——保留了空间细节但**所有位置都保留**(无选择性);#5 的 P2+熵稀疏化是选择性保留(只在前景区)。这是"稠密传递 vs 稀疏传递"的路线分歧。
3. **为什么没人做?** YOLO-DETR 混合架构刚出现,社区还在探索 backbone+neck 的最优组合,未深入到稀疏化。
4. **如果我是作者,下一篇做什么?** PBM + 显式 P2 头的叠加收益分析;PBM 的选择性版本(只传高信息量捷径——即 #5 的思路)。
5. **新 Gap**:PBM 在纯 CNN backbone(非 ViT)上的有效性未知——CNN 的浅层特征与 ViT 的 token 在信息密度上不同,PBM 可能需要在 CNN 上重新设计。

### 来自 DERNet (arXiv 2026.06) —— ⚠️ 已由 P0 深读取代 (2026-07-19), 详见下方 🟡🔬 条目
1. **别人没解决什么?** 频域处理只在特征增强维度,从未用于**计算资源分配**。DERNet 的 WDG/LGE/FDHead 全部做"频域→更好的特征",不做"频域→决定要不要算"。
2. **瓶颈**:小波/Log-Gabor 在 GPU 上的 wall-clock time 可能显著高于理论 FLOPs(小波变换是 memory-bound 操作)。P2 分辨率是 P5 的 16 倍——DERNet 的频域处理在 P2 上的实际速度是未知数。
3. **为什么没人做?** 频域+检测是 2025–2026 才热的方向,目前大家都在探索"怎么用频域提升特征质量",还没人想到"用频域省算力"。
4. **如果我是作者,下一篇做什么?** 频域引导的自适应计算分配(高频区域→精细处理,低频区域→粗处理)——但 DERNet 自己还没走到这步,#11 可以抢先。
5. **"1/6 参数超越 YOLOv11" 的背面**:参数少≠算力少——小波变换工程效率是关键限制。

### 来自 SET (CVPR 2025) —— ⚠️ 对 #11 的根本性质疑
1. **别人没解决什么?** "移除高频→帮助小目标"的反直觉发现被提出,但**这个结论在 P2 层是否成立不知道**——P2 未经多次下采样,高频噪声累积少,结论可能反转。
2. **瓶颈**:HBS 依赖精确 GT box mask——小目标标注噪声(VisDrone 常见)下 HBS 可能误平滑前景边缘。
3. **为什么没人做?** 频域分析在检测中刚起步,大家都从 FPN 中间层开始分析,P2(最高分辨率)的计算开销让频域分析望而却步。
4. **如果我是作者,下一篇做什么?** 自适应频谱策略:对极小目标抑制高频噪声(P2),对大目标保留高频细节(P5)——尺度特异的频域处理。这正是 #11 可以做的(高频判据+尺度自适应)。
5. **#11 的修正方案**:SET 证明"绝对高频能量"作判据有问题→应改用**"高频能量局部异常度"**(小目标高频在空间上点状孤立,背景高频规则分布)。

### 来自 D3Q (IEEE JSTARS 2025) —— DETR 线小目标能力定位
1. **别人没解决什么?** DETR 的动态查询解决了"查询数量 vs 目标密度"的矛盾,但**小目标特征质量本身**(token 坍缩、空间退化)没改善——D3Q 是"给更多查询去搜索",而非"让每个查询看得更清"。
2. **瓶颈**:动态查询依赖密度估计(需 GT 监督)——无监督/自监督密度估计(用熵/频域先验)可替代但无人尝试。
3. **为什么没人做?** DETR 社区在解决收敛速度/训练效率(DINO/DN-DETR/DAB-DETR),小目标专项优化留给了 CNN 系。
4. **如果我是作者,下一篇做什么?** D3Q 的密度估计用频域/熵先验替代 GT 监督→零标注开销的查询数量自适应。
5. **对"坚持 YOLO"决策的影响**:D3Q(49M/543G)vs YOLO11n(2.6M/6.3G)——DETR 在 VisDrone 上无压倒性优势,轻量/实时场景 YOLO 不可替代。**决策不变。**

### 来自 SFDNet (ECCV 2026) —— 频谱解耦分治法
1. **别人没解决什么?** 三频解耦(低/中/高)比单一高频判据精细,但 SFDNet 只做特征增强——**频谱信息用于条件计算/稀疏化的潜力完全未探索**。
2. **瓶颈**:三路处理的计算开销(低/中/高三条并行通路)可能抵消频谱解耦的优势;CPD 原型蒸馏在类别数大的场景(COCO 80 类)效果未知。
3. **为什么没人做?** 频域分治 + 原型蒸馏是新的技术组合,刚被提出,社区还在消化。
4. **如果我是作者,下一篇做什么?** ASD 的频谱权重用熵/频域先验替代学习(免训练→零参数→更轻量),原型蒸馏用 CLIP 文本原型替代视觉原型(零 shot)。
5. **#11 的升级路线**:从"高频 vs 低频"的二元判据→多频谱能量分布作向量判据(每个位置的频谱签名)。

### ⚠️ 频域小目标浪潮: 3篇→4篇 (2026-07-16 更新)

**现象**:至少 4 篇独立工作(SET CVPR 2025 + FMC-DETR arXiv 2025.09 + DERNet arXiv 2026.06 + SFDNet ECCV 2026)在同一时间窗口内瞄准"频域+小目标检测",且彼此无相互引用(投稿时间线重叠)。**频域小目标检测正处于早期快速上升期。**

**四篇的共同点**:
- 都承认频域信息对小目标定位至关重要
- 都用频域做**特征增强**(更好的特征→更准的检测)
- 都有即插即用设计

**四篇都没做的事(这正是 #11 的位置)**:
- 用频域信息做**条件计算/稀疏化**(频域→判定重要性→跳过冗余计算)
- 在 **P2 层**专门做频域分析(P2 分辨率太高,四篇都回避了)
- 将频域判据与**语义判据**(CLIP 熵)对比/融合

**#11 的差异化生命线**:Related Work 引这四篇,承认频域对小目标有效,然后说"但现有工作只在增强维度使用频域,其作为**计算资源分配依据**的潜力未被探索——本文首次提出频域引导的特征级条件计算,在保持小目标精度的同时大幅降低 P2 头的算力开销。"

**叙事策略建议**:
- #11 的标题/摘要必须出现 "frequency-guided conditional computation" 或 "spectral sparsification" 以与"frequency enhancement"路线区隔
- Related Work 需显式划界:"不同于 SET/FMC-DETR/DERNet/SFDNet 利用频谱改善特征质量,本文利用频谱改善计算效率"

### 来自 FMC-DETR (arXiv 2025.09) —— 振幅-相位解耦 × 条件计算

1. **别人没解决什么?** ①MDFC的振幅-相位解耦极优雅(振幅="改多少",相位="在哪"),但全图统一调制无空间选择性;②WeKat的小波递归分解(HSG-WAVE)在浅层扩大ERF+抑制噪声,但小波特征未被用于判据;③DETR侧也发现D2(浅层高分辨率)关键,验证了P2/D2的跨架构价值。
2. **瓶颈**: DETR-based,FPS仅17.8;小波GPU墙钟效率未消融;MDFC三路并行+CPF四分支的计算开销。
3. **为什么没人做?** 频域+检测社区正探索"怎么用频域提特征质量",未想到"用频域省算力"。
4. **如果我是作者,下一篇做什么?** ①振幅引导的空间选择性FFT(高频区域→精细频域分析)→这正是#11;②YOLO版轻量MDFC→YOLO head。
5. **新交叉 Gap: 振幅调制 × 语义熵 × 高频判据**——三种门控范式(频域振幅/语义熵/高频能量)功能同构但信息来源互补,可AND/OR融合。

### ⚠️ 对#11叙事的影响: 4篇Related Work证据链完整

Related Work更加充实——"已有4篇独立工作(SET/FMC-DETR/DERNet/SFDNet)共同证明频域信息对小目标检测至关重要,但它们仅在特征增强维度利用频域。本文首次将频域信息用作计算资源分配的依据..."

### 来自 ViCrop-Det (arXiv 2026.04) —— 注意力熵 × 语义熵 交叉路口

1. **别人没解决什么?** ①SAE是DETR专属(YOLO无cross-attn decoder),注意力熵范式在单阶段检测器上完全空白;②零注意力盲区(模型完全不关注的目标)是SAE的硬伤,VLM语义先验可覆盖此类盲区;③SAE仅用于推理时裁剪路由,从未被建模为模型内部的特征门控信号。
2. **瓶颈**: YOLO没有cross-attention→SAE无法直接移植;但YOLO可以用**语义熵**(来自VLM)或**特征能量熵**(免VLM)作为替代判据。
3. **为什么没人做?** YOLO社区未接触注意力熵概念(ViCrop-Det也是DETR专属);VLM社区不做特征级稀疏化。
4. **如果我是作者,下一篇做什么?** ①将SAE替换为语义熵(CLIP嵌入熵)→覆盖零注意力盲区→提升全面性;②将裁剪路由替换为特征级门控→嵌入模型内部→减少推理开销。
5. **新 Gap: 注意力熵×语义熵双源门控的交叉点**——两者功能同构但信息来源互补(模型内生 vs 外部知识),AND/OR融合即可超越单一判据。**这直接强化Idea#5的叙事**,且ViCrop-Det可作正交对比引用。

### 来自 D³R-DETR (arXiv 2026.01) —— Gabor 频域核 × 密度稀疏化

1. **双 gap 均维持**: ①**频域条件计算空白第 7 次确认**——FSDETR(IJCNN 2026,可学习 2D-DFT 频域滤波)为频域 DETR 浪潮第 7 篇,仍纯增强范式,"频谱判据→token 条件计算"(#30)无人进入;②**P2 内空间条件计算空白第 N+2 次确认**——本轮 YOLO P2 侧新作(SDD-YOLO/HFSP-YOLO/HierLight-YOLO)仍全部为"加 P2 头/注入 P3"路线;HFSP-YOLO 的 Space-to-Depth 注入 = P2 利用**第三路线**(加头/头内稀疏/注入),但仍是静态结构,无输入自适应 → #5 gap 稳固。
2. **Unmasking the Tiny 哨点降级(见刊裁决)**: IVC Vol.172 细节证实 STSM+FRM 为**补强式(加法)**——粗选弱信号 token 后用语义注意力恢复被抑制的前景分数;与 #5 **跳过式(减法)**方向相反,同一特征图可共存 → 上一条"最近哨点"警报解除,降级普通近邻(其"转向省算力"的风险路径被其自身机制方向否定);三轴划界定稿:目的(召回恢复 vs 算力分配)/信号(自身前景分数 vs VLM 语义熵)/机制方向(加法 vs 减法)。
3. **条件计算维度地图补全(HI-MoE)**: DETR 侧条件计算现有两个维度——**专家/容量维**(HI-MoE 两级 MoE,+3.3 APs)与**判据引导 token 空间维**(Dome);#30 的免监督频谱判据落在后者且判据免监督 → 与两者均划清;"条件计算利好小目标"获两个独立印证 = 通路侧翼佐证。
4. **熵引导的第三种用法出现(MFVL-YOLO)**: 熵引导已有三用法——裁剪路由(ViCrop-Det)/前景判别增强(MFVL-YOLO)/**空间计算分配(#5,仍空白)**;#5 概念红线表述「熵引导的空间计算分配」价值再显。
5. **#7 消融对照轴现成(Scale-Conscious KD)**: 面积加权蒸馏(SAOD)直击"大目标主导蒸馏梯度"——与 #7 语义熵加权构成"几何先验 vs 语义不确定性"天然对照消融,写论文可直接引用设计。

### PRNet 深读增量(2026-07-18,N+3 次确认)

1. **P2 复用"值+代价"矛盾的量化暴露(#5 motivation 最强新证)**: PRN 消融链完美演示——P2^in 原始信息被 FPN 只用一次是大浪费(+10.3 AP50),但缠密复用代价直观(FLOPs +110.7%) → 选择性复用的必要性不再只是理论推演,而是有确凿数字背书。与 Edge-Constrained "P2 alone +31% AP_small" 并列 #5 叙事的最强动机数据点。

2. **P2 路线图第三→第四分支确立**: HFSP-YOLO 补充了"注入 P3"第三路线,PRNet 补充了"骨干 P2 复用+迭代精炼"第四路线。四条路线(加头/注入/粗粒度选择/骨干复用)全部静态稠密,无输入自适应 → **#5 gap = 跨路线共性空白**(不是仅某一支路的空白)。

3. **层深维条件计算的新入口(PRN 迭代深度)**: PRN 的阶段 0/1/2/3 = 45.0/49.3/51.0/51.4 AP50,收益递增但 FLOPs 线性涨(28.7→67.5G),作者一刀切取 1——与 #26(decoder 层深最优停止)构成跨轨"层深维条件计算": 用无监督判据(高频统计/熵)按图像复杂度自适应决定精炼深度。与 #5 的空间维互补(一横一纵)。

4. **DETR 增益缩水 = 架构差异的定量锚点**: PRN 在 YOLO 系(+6~10)vs RT-DETR(+3.2)的增益差,为 #24 信息瓶颈叙事提供"CNN 复用原始细节比 self-attention 更契合浅层信息保持"的实证数据点,写论文时可反面论证 B轨需独立设计。

5. **PRNet 无 FPS 漏洞 = #30 E6 设计正面理由**: PRNet 一文自称 real-time 但全文无 FPS,仅 FLOPs 作代理——强化 #30 实验协议 E6"报 GFLOPs 分布+FPS+延迟"的必要性,审稿若被问"为什么额外报这些"可引 PRNet 反例。

## 🔴 K1 Gap 分析: 密集遮挡方向系统化缺口（2026-07-18）

> 基于 L1→L3 全部 11 篇密集遮挡论文（DALA/OPL/FAFL/DOMino-YOLO/DRONet/HEdge-MamYOLO/MFF-YOLO/GCS-DETR/RepLoss/CrowdDet/Soft-NMS），按 TASKS 指定四维度系统分析。每个 Gap 标注严重程度（🔴致命/🟠重要/🟡机会）、可验证性（⏸需实验/✅可文档推演）、关联 Idea。

### 一、标签分配 (Label Assignment) 在密集场景

#### K1-G1 🔴 密度感知 LA 的粒度断层：硬二分类→连续软值
- **现状**: DALA 是首个密度感知 LA，但密度分类是**硬二分类**（τ_dense 阈值，密集/稀疏两档）。实际场景中密度是连续谱——VisDrone 停车场（极度密集）、道路交叉口（中度密集）、天空（稀疏）三者需要不同策略。
- **为什么是 Gap**: 硬二分类导致阈值附近的目标分配策略突变（τ 左侧 O2O、右侧 Decreasing LA）→ 训练不稳定 + 边界目标分配次优。连续软密度值→连续分配策略过渡是更自然的方案，但无人实现。
- **可验证性**: ⏸ 需实验（DALA 代码 + VisDrone 消融：硬分类 vs 连续软密度值）
- **关联 Idea**: #31（密度×频域联合判据）、已有 DALA improvement #1

#### K1-G2 🟠 密度定义的维度单一：纯空间→多维正交
- **现状**: DALA 的密度 = GT 邻域距离（纯空间度量）。但空间近≠语义混淆——空间近但纹理/颜色不同的目标可被特征区分，不需要 O2O；空间远但语义混淆（同类密集）的目标反而需要集合预测去重。
- **为什么是 Gap**: 空间密度 + 频域密度（高频局部统计）+ 语义密度（熵相关）→三维正交密度定义 → 更准确判断哪些目标真正需要 O2O/EMD。频域判据在此可提供**与空间密度正交的语义密度信息**。
- **可验证性**: ⏸ 需实验（频域图局部统计 vs GT 空间密度的 Spearman ρ 相关性分析，可 CPU 执行但暂缓）
- **关联 Idea**: #31（频域×密度联合）、#35（频域遮挡先验）

#### K1-G3 🟡 衰减 schedule 的数据驱动化
- **现状**: DALA Decreasing LA 的衰减 `K(t) = max(1, K_max · decay(t))` 是**纯 epoch 驱动**（线性/余弦），与样本实际训练状态无关。稀疏目标可能在 epoch 10 已收敛→继续 K>1 引入噪声；难例目标 epoch 50 仍未收敛→K 已衰减到 1→表征不足。
- **为什么是 Gap**: 数据驱动衰减（如 loss 收敛速度/梯度范数→动态调整 K）可以按样本粒度自适应，而非全数据集统一 schedule。与 #5 门控同源——都是"何时停止/跳过"的决策问题。
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #5（语义熵门控共享"何时跳过"哲学）、#26（最优停止理论）

#### K1-G4 🟠 标签分配×特征条件计算的协同框架空白
- **现状**: DALA 的"密集→O2O→正样本精简"控制训练信号分布，#5/#11 的"高熵/低频→P2 token 跳过"控制推理计算分布。两者是**条件计算的两个独立维度**（训练信号 vs 推理计算），但从未在同一框架下联合设计。
- **为什么是 Gap**: 双维协同理论上可产生乘法式收益——密集区域同时做 O2O（减训练噪声）+ P2 全计算（保留精度），稀疏区域同时做 Decreasing LA（充分表征）+ P2 高稀疏（省算力）。**无任何已有工作触及此交叉点**。
- **可验证性**: ✅ 理论推演可完成（四象限策略矩阵：dense/sparse × high/low freq → 联合策略），实验验证⏸
- **关联 Idea**: #31（密度×频域双维条件计算——**此 Gap 的直接回应**）

### 二、NMS 敏感性

#### K1-G5 🔴 NMS 密度自适应衰减：全局静态 σ→局部动态
- **现状**: 所有 NMS 变体（传统 NMS/Soft-NMS/NWD-Soft-NMS）的衰减参数（σ/N_t）是**全局静态**的。密集区域需要更大的 σ（更宽容→保留真实目标），稀疏区域需要更小的 σ（更严格→消除冗余框）。全局 σ 是折中→两端都不最优。
- **为什么是 Gap**: 密度自适应 σ 从 2017 年 Soft-NMS 提出至今近 10 年无人做。实现成本低（局部目标计数→σ 查表或线性映射），收益直观（密集区召回↑、稀疏区精度↑）。是"低成本高回报"的典型被忽视方向。
- **可验证性**: ⏸ 需实验（VisDrone 消融：全局 σ vs 密度自适应 σ vs per-image σ）
- **关联 Idea**: #38（频谱感知 Soft-NMS，频域内容相似度→NMS 第二判据——可与此 Gap 合并为"密度+内容双自适应 NMS"）

#### K1-G6 🟠 NMS 内容感知衰减：坐标重叠→内容相似度
- **现状**: 所有 NMS 变体仅用框坐标（IoU/NWD）判断冗余，完全忽略框内特征内容。两个重叠 90% 的框如果内容完全不同（不同类别/不同实例的不同纹理），不应衰减。反之，两个重叠 30% 但内容完全相同的框（同实例重复检测），应衰减。
- **为什么是 Gap**: NMS 从诞生至今只考虑"空间重叠≈冗余"，但密集遮挡场景下空间重叠与内容冗余的对应关系被打破。频域判据（框内频谱统计→余弦相似度）提供与 IoU 正交的内容信号。
- **可验证性**: ⏸ 需实验（框内频谱相似度 vs 是否同实例的 GT 标注→AUROC 验证，类似 #30 E3 模式）
- **关联 Idea**: #38（**频谱感知 Soft-NMS**——此 Gap 的直接回应）

#### K1-G7 🟡 NMS 可学习衰减函数
- **现状**: Soft-NMS 衰减函数 `f(IoU) = 1 - IoU` 或 `exp(-IoU²/σ)` 是手工设计的，非数据驱动。最优衰减形状因数据集/场景/密度而异——小目标密集场景可能需要"高重叠时缓慢衰减、低重叠时快速衰减"的不对称函数。
- **为什么是 Gap**: 用小型 MLP 学习 `f(IoU, NWD, 频谱相似度, 局部密度) → 衰减系数` 可端到端优化，但 NMS 的非可微性（贪心选择→离散操作）是技术障碍。Gumbel-Softmax 近似 + 强化学习是可能路线。
- **可验证性**: ⏸ 需实验（技术难度高，低优先级）
- **关联 Idea**: #38 远期扩展

#### K1-G8 🟡 训练期排斥 vs 推理期 NMS 的联合优化
- **现状**: RepLoss（训练期推开预测框）+ Soft-NMS（推理期衰减冗余框）互补但**从未在同一检测器上联合使用**。两个阶段的参数（α/β/σ）独立调优，无法协同。
- **为什么是 Gap**: 联合优化可能实现"训练期轻度排斥 + 推理期轻度衰减"→两端都可以更保守→减少各自引入的精度损失。核心挑战是 NMS 不可微→需近似或交替优化。
- **可验证性**: ✅ 概念推演可完成（三阶段覆盖分析已在 loss.md 完成），实验验证⏸
- **关联 Idea**: #39（EMD+RepLoss 联合，#37 先行→训练期匹配+回归联合），加上 Soft-NMS→"训练-推理全链路密集优化"

### 三、遮挡特征恢复

#### K1-G9 🔴 遮挡先验来源的"最后一公里"：从标注依赖到免标注自感知
- **现状**: 遮挡先验的四种来源——①人工标注（OAR-Loss 5级、RepLoss GT）②自动 GT（OPL bbox重叠+高斯模糊）③可学习（Dome DeFE 密度头+DRFL）④**物理先验（频域/熵→#35/#36，空白）**。前三者都有依赖（标注/重叠假阳性/GT监督），第四种（免标注+无重叠假设+覆盖截断/背景遮挡）是范式跃迁。
- **为什么是 Gap**: 频域判据（高频局部异常度→边缘突变=遮挡边界）覆盖 bbox 重叠无法捕获的场景：截断遮挡（目标被图像边缘切掉）、背景遮挡（被非目标的建筑物/树遮挡）、完全遮挡（无 bbox 的目标）。从"需要人告诉模型什么是遮挡"到"模型从信号本身感知遮挡"→遮挡建模的终极形态。
- **可验证性**: ⏸ 需实验（频域图 vs OPL 遮挡 GT/人工标注→AUROC，可 CPU 执行但暂缓）
- **关联 Idea**: #35（频域遮挡先验——**此 Gap 的直接回应**）、#36（语义熵隐式遮挡检测器）

#### K1-G10 🟠 "增强遮挡区" vs "跳过高熵区" 的张力裁决
- **现状**: OPL/FAFL/OAR-Loss 对遮挡区**增强**（注入补偿信息），#5 对高熵区**跳过/降权**（高熵≈不可靠→降低计算）。两者的方向相反，根源在于对"遮挡区域高熵成因"的不同假设——增强派认为"特征被邻域干扰→需要补全"，跳过派认为"背景冗余信号→不值得算"。
- **为什么是 Gap**: 裁决这个张力需要回答一个核心问题：**遮挡区域的高熵是因为特征弱（需要增强）还是语义混合（需要跳过）？** 答案决定了"增强→跳过"全链路的正确次序。调和方案：先熵过滤背景（#5）→再在保留的前景区做遮挡增强（OPL-like）。
- **可验证性**: ⏸ 需实验（VisDrone 遮挡区 vs 非遮挡区的语义熵分布统计→若遮挡区熵显著偏高且方差大→支持"先过滤后增强"管线）
- **关联 Idea**: #36（语义熵-遮挡相关验证——**此 Gap 的直接裁决实验**）

#### K1-G11 🟠 频域特征修复：从增强到补全
- **现状**: HEdge-MamYOLO FM-CHFEM 证明"频域提取→全局搜索→特征修复"三步管线有效（VisDrone 52.5%），但整体依赖 Mamba backbone→CNN 等价实现是 YOLO 迁移的关键缺失。
- **为什么是 Gap**: #11 当前仅用频域做门控（"算不算"），FM-CHFEM 证明频域也可以做修复（"怎么算得更好"）。"判据+修复"统一框架（同一频域图→同时决定门控决策+修复强度）是频率信息的最大化利用。
- **可验证性**: ⏸ 需实验（Mamba→CNN 等价架构搜索）
- **关联 Idea**: #11 v2.0 远期方向

#### K1-G12 🟡 检测头通道维×空间维双维门控
- **现状**: CSIM-Head（通道维→抑制遮挡噪声通道）+ #5 P2 门控（空间维→跳过背景位置）两者互补但从未在同一检测器上协同。通道门控 + 空间门控 = 双维联合稀疏化。
- **为什么是 Gap**: 遮挡区域的"哪些通道不可靠"（CSIM）和"哪些位置不可靠"（#5）是正交问题→联合解决可覆盖更多遮挡失败模式。
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #5 v3.1（空间+通道双维稀疏扩展）、compare.md 结论 33⑤

### 四、DETR Query 冲突

> ⚠️ 本维度受 2026-07-18 策略修订影响：DETR 降级为交叉融合副线，纯 DETR query 机制分析降低优先级。以下 Gap 仅保留**可向 YOLO 迁移概念/判据**的部分。

#### K1-G13 🟠 密度自适应 query 配额的免监督实现
- **现状**: Dome PAQI（GT 密度图+DRFL 监督→弹性 query 数量）、D3Q DQS（密度分类→4 档 query 数）、DQ-DETR CCM（密度预测→自适应 query 数）全部依赖 GT 监督或手工密度分档。**免监督密度判据→query 配额**完全空白。
- **为什么是 Gap（YOLO 迁移视角）**: YOLO 的 grid-cell 级预测密度（P2 每 cell 预测几个实例）与 DETR 的 query 配额问题是同构的——密集 cell 需要更多预测。频域/熵判据可作为 YOLO grid-cell 的"局部密度指示器"。概念可迁移，实现不同。
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #34（密度自适应聚合窗口，概念层可迁移 YOLO）、#37（YOLO Grid-Cell EMD）

#### K1-G14 🟡 局部集合预测（CrowdDet EMD）× 全局二分匹配（DETR Hungarian）的层次化
- **现状**: CrowdDet EMD 在 proposal 级做局部集合预测（K=2），DETR Hungarian 在图像级做全局集合预测（300 query）。两者从未在同一框架下层次化组合——局部 EMD 解决 proposal/grid-cell 内多实例，全局 Hungarian 解决跨 cell 重复。
- **为什么是 Gap（YOLO 迁移视角）**: YOLO 的 grid-cell 结构天然适合局部集合预测（P2 cell 小→密集区自然多→K=2-3），而跨 cell 重复可通过 Set-NMS-like 后处理或全局二分匹配解决。局部+全局的两层集合预测是 YOLO DETR 融合的潜在突破口。
- **可验证性**: ✅ 概念推演可完成，实验验证⏸
- **关联 Idea**: #37（YOLO Grid-Cell EMD）+ #39（EMD+RepLoss 联合）

#### K1-G15 🟡 端到端 NMS-free 密集检测的 YOLO 路径
- **现状**: DETR 的端到端（Hungarian 匹配→NMS-free）在密集场景仍需 NMS（Dome 保留 Dynamic NMS、D³R-DETR 同理）。YOLO 侧 DALA 的 O2O LA 也瞄准 NMS-free。但两者都没彻底解决——**密集遮挡下的真 NMS-free 检测仍是开放问题**。

### 来自 DALA (ESWA 2026) —— 密度感知标签分配 × 频域判据空白

1. **别人没解决什么?** DALA 是首个将密度显式建模进标签分配的工作，但其密度定义是**纯空间域**的（GT 邻域距离）→频域信息（高频能量/频谱差异）作为密度的正交度量**完全未被探索**。此外，DALA 的衰减 schedule 是**静态 epoch 驱动**（非数据驱动），密度分类是**硬二分类**（无软过渡）。

2. **瓶颈**: 
   - 空间密度 ≠ 语义密度——空间近但语义差异大的目标（如不同颜色的车）在 DALA 中被误判为密集，但实际不需要 O2O（特征可区分）
   - 密度阈值 τ_dense 需数据集特化，无自适应机制
   - 仅 CNN 检测器验证，DETR 移植空白

3. **为什么没人做?** 密度感知 LA 2026 年才首次被提出，社区仍在消化"标签分配需要密度感知"这个概念→"密度感知 LA + 频域判据"自然是下一层未触及的组合

4. **如果我是作者,下一篇做什么?** 
   - 密度定义升级: 空间密度 + 频域密度（高频能量图局部统计）→联合判据
   - 衰减 schedule 数据驱动: 用稀疏目标的训练收敛状态（loss 变化率/梯度范数）动态决定衰减时机
   - DETR 移植: 密度分类 → query 配额自适应（密集 GT→1 query，稀疏 GT→K_max→1）

5. **新 Gap 确认: 密度感知 LA + 频域判据的组合空白**——DALA 的密度是空间域（GT 距离），频域判据（高频能量/频谱差异）提供语义密度的正交信息→两者组合可同时区分"空间密集但语义可分（高频判决可区分）"与"空间密集且语义混淆（高频判决不可区分）"→后一类才是真正需要 O2O 的目标。这个 Gap 同时跨越 A 轨（DALA + #5/#11 CNN 条件计算）和 B 轨（DALA + #30 DETR token 条件计算），是**双轨共享的跨域空白**。

6. **新 Gap: 标签分配条件计算 × 特征条件计算的协同框架空白**——DALA 的"密集→O2O→正样本精简化"和 #5/#11 的"高熵/低频→P2 token 跳过→特征计算稀疏化"是**两个独立维度的条件计算**: 前者控制训练信号分布，后者控制推理计算分布。将两者纳入统一框架（密度×熵/频域联合判据→同时决定正样本数和 token 计算量）是全新的理论方向，**无任何已有工作触及**。

### 来自 OPL (ESWA 2025) —— 显式遮挡感知 × 频域判据替代手工遮挡图生成

1. **别人没解决什么?** OPL 是首个显式遮挡感知损失（OPD + OPL + OPC），但其 GT 遮挡图生成依赖 **bbox 重叠 + 高斯模糊**——这有三个根本局限：①bbox 重叠 ≠ 真实遮挡（并排无遮挡行人也被标记）；②完全被遮挡目标无 bbox→无法生成 GT 遮挡图；③遮挡图只在训练时生成，推理时无在线遮挡感知能力。**频域判据（高频局部异常度）可以替代 bbox 重叠作为遮挡先验**，且不依赖多目标交叠→覆盖更广泛的遮挡类型。

2. **瓶颈**:
   - 频域判据"看到"的是高频突变（边缘/纹理），不一定是语义遮挡→需要验证频域图与真实遮挡的相关性（AUROC vs OPL 遮挡 GT）
   - OPL 的 OPD 需要训练监督信号，频域判据替代 GT 后的监督质量需实验验证
   - OPL 仅验证行人检测（CrowdHuman/CityPersons），VisDrone 无人机的密集遮挡场景效果未知

3. **为什么没人做?** 遮挡感知 Loss 和频域判据是两条独立的文献线——OPL 来自行人检测社区、频域判据（#30）来自轻量化/条件计算社区。两条线从未交叉："遮挡图生成免 bbox 重叠依赖"的思路在两社区交汇之前不可能出现。

4. **如果我是作者,下一篇做什么?**
   - **遮挡图生成的频域替代方案**: 高频局部异常度图 + 自适应阈值 → 替代 bbox 重叠的 GT 遮挡图 → 覆盖截断/背景遮挡/密集人群
   - **熵图 = 隐式遮挡检测器**: 遮挡区域天然高熵（混合特征→语义不纯）→ 验证 OPL 遮挡图 vs 语义熵图的空间相关性 → 若 Spearman ρ > 0.6，语义熵可作零参数遮挡先验
   - **DETR 化 OPL**: per-query 遮挡图 + Cross-Attention bias 注入→更精细遮挡建模 + 免 NMS

5. **新 Gap 确认: 免 bbox 重叠的遮挡图生成**——OPL 证明了"显式遮挡建模有效"，但其 GT 生成依赖 bbox 重叠。频域/熵判据提供了一条**完全免标注、无 bbox 重叠假设**的替代路线。这个 Gap 同时关联：
   - 🟦 A轨: 语义熵图（#5）是否隐含遮挡信息？→ 验证可让 #5 叙事从"条件计算"扩展到"遮挡感知计算分配"
   - 🟪 B轨: 频域判据（#30）是否可作为 DETR 的遮挡感知先验？→ #30 应用维度从 token 预算扩展到遮挡增强
   - 🔴 密集遮挡: 首个连接"遮挡建模"和"条件计算"的双轨交汇点

6. **OPL 与 #5 的关键张力**: OPL 对遮挡区域**增强**（注入遮挡图补偿信息缺失），#5 对高熵区域**跳过/降权**（高熵≈不可靠→降低计算投入）。这一张力需实验裁决：遮挡区域的高熵是因为"特征弱"还是"语义混合"？若是前者（特征弱→高熵），增强有理（OPL 立场）；若是后者（语义纯但被邻域干扰），跳过有理（#5 立场）。**调和方案**: 先做熵门控过滤背景（#5 本体），再在保留的前景区域做遮挡增强（OPL 借用的 OPC 类似物）→ 两层过滤 + 一层增强。

### 来自 Dynamic DETR (ICML 2025) —— 🔬深读 2026-07-18,#30 判据对照空白正式确立

1. **别人没解决什么?** Dynamic DETR 解决了"token 重要性跨层迁移→动态保留率"的架构级适配，但其判据(DTA)本质是 **attention weight 在 COCO val-set 上的离线统计先验**——每张输入图共享同一套 stage 级保留率排序，**输入级自适应完全未做**。频谱/梯度等物理免训练判据零涉及。且 ρ 向量需每模型手动扫参（Table 6 五组配置），换数据集/模型须重新统计。为什么: 作者的出发点是"发现规律→固化为先验"，而非"每图实时判断"——统计先验路线在 COCO 这种分布均匀的基准上够用，掩盖了输入自适应的需求。

2. **瓶颈**:
   - 技术层面: 离线统计在分布外场景(航拍/密集小目标)失效风险——VisDrone 中浅层重要性可能持续到深层 block，"低→高迁移"假设未必成立
   - 评估层面: COCO/LVIS/VOC 全是常规尺度基准，token 稀疏化方法在小目标专项基准(VisDrone/AI-TOD)上的表现是系统性盲区(Sparse/Focus/Lite/Dynamic 全都没测)
   - Proximal Aggregation 的 n²→1 合并本质是空间下采样，对 ~10px 目标的定位影响无消融

3. **为什么没人做?** ICML 社区的效率工作以 COCO 为默认战场，小目标社区(RS/UAV)与 token 稀疏化社区几乎零交集; 免训练物理判据(频谱)属于"低技术含量"表象，顶会更偏好可学习机制——这恰是 #30 的逆向机会。

4. **如果我是作者,下一篇做什么?** (a) 把 DTA 统计先验升级为逐图在线判据(轻量频谱统计是天然候选); (b) stage 划分 N1/N2/N3 自适应化; (c) 在 VisDrone 验证重要性迁移规律是否翻转。

5. **#30 判据对照空白正式确立**: 深读裁定 Dynamic DETR 与 #30 **不撞车**(判据来源/自适应粒度/泛化性三轴分离,详见 [Summary §8.4](../papers/summaries/Dynamic-DETR_ICML2025.md))，且形成互补——**「学习式/统计先验判据(Dynamic DETR) vs 免监督物理判据(#30 S1 空域高通)」的头对头对照现在有了顶会级 SOTA 基线**。#30 的 E1 实验应新增 Dynamic DETR 对照列: 同一稀疏率下比较 attention 统计先验 vs 频谱判据的 AP-效率曲线，并消融"Dynamic DETR 框架 + #30 判据"的混合配置(其 MTA 双轨聚合与 RCDR 正则可直接复用)。**新增红利**: 其 RCDR(中心距离正则,+0.6 AP 零开销)是判据无关的通用组件，#30 可直接引入。

6. **新 Gap: token 稀疏化 × 小目标基准的系统性空缺**——全部四个 token 稀疏化方法(Sparse/Focus/Lite/Dynamic)都只测 COCO 系基准。谁第一个在 VisDrone/AI-TOD 上系统评估 token 稀疏化(并解决小目标 token 被误合并问题)，谁就占据"efficient DETR for small objects"生态位——这正是 #30 的战场，也是新 Idea #33(频谱判据×MTA框架)/#34(密度自适应聚合窗口)的出发点。

### 来自 L1.5 P1 密集遮挡5篇深读 (2026-07-18)

#### DOMino-YOLO (MDPI RS 2025)

1. **别人没解决什么?** Repulsion Loss 自 CVPR 2018 以来从未引入遮挡可见度维度——OAR-Loss 首次将遮挡等级量化纳入排斥损失权重，但需要人工遮挡标注(VOD-UAV 5级标注)。**免标注遮挡先验替代方案完全空白**——频域判据/语义熵可作为零标注遮挡权重源。

2. **瓶颈**: VOD-UAV 数据集仅含车辆类→多类别密集场景(VisDrone 10类)的泛化性未知; OAR-Loss 的可见度权重依赖 GT 标注→实际应用标注成本高; VASA 和 DCEM 可能功能重叠(都解决"遮挡→空间信息损失")。

3. **为什么没人做?** 遮挡感知损失方向本身还很新(OPL 2025 首个显式遮挡损失 + OAR-Loss 2025 首个遮挡 RepLoss)，标注依赖是"先有标注才能做"的鸡生蛋问题——频域判据/熵判据恰好可以打破这个循环。

4. **如果我是作者,下一篇做什么?** 用频域高频能量/语义熵作为免标注遮挡权重替代→使 OAR-Loss 可泛化到任意无遮挡标注的数据集。

5. **新 Gap**: 类条件排斥损失——不同类间遮挡 vs 同类密集的排斥力应不同，当前所有 RepLoss 变体同类异类无差别排斥。

#### DRONet (Displays 2026)

1. **别人没解决什么?** KAN 在目标检测 backbone 的应用几乎空白——DRONet 是首批实践者，但 KAN 非标准 CNN 组件→ultralytics 集成困难。**标准 CNN 等价实现 KAN 的频率多样性**是开放问题——DCT 基核组+可学习权重能否达到相同效果?

2. **瓶颈**: OAKB 仅在 ResNet18 验证→更大 backbone(R50/R101)上的可扩展性未知; CARPK +0.7% vs VisDrone +3.1%→KAN 价值高度绑定不规则遮挡场景; PSI/SDEA/OAKB 各自贡献无消融数据。

3. **为什么没人做?** KAN 在 CV 社区仍属边缘工具，主流 backbone 设计以 CNN/ViT/SSM 三分天下; 频域参数化技术(GPU 频域核)跨学科门槛高。

4. **如果我是作者,下一篇做什么?** 将 OAKB 的频域参数化思想应用于 #11 S1 判据的高效实现(Sobel/LoG→频域核等效)。

5. **新 Gap**: "检测→修复→补全"三层遮挡处理管线——DRONet 隐式实现了 Backbone(OAKB 判别遮挡碎片)→Neck-浅(PSI 选择性聚合)→Neck-深(SDEA 多尺度补全)，但未显式概念化。显式设计三层管线并引入频域判据在各层做自适应决策。

#### HEdge-MamYOLO (IEEE TGRS 2026)

1. **别人没解决什么?** 频域方法在目标检测中全部停留在"增强"范式(SET/DERNet/FMC-DETR等7+篇)——FM-CHFEM 是首个"频域提取→全局搜索→特征修复"的**修复范式**。但 Mamba backbone 生态不成熟→**CNN 等价实现是 YOLO 迁移的关键缺失环节**。

2. **瓶颈**: VisDrone 52.5% 是检索所见最高，但 Mamba backbone 贡献 vs FM-CHFEM 贡献无消融→若 backbone 是主因则迁移价值受限; 三步流水线(频域变换+SS2D+空间插值)的计算开销未公开; 频域变换具体工具(DCT/FFT/Wavelet)未确认。

3. **为什么没人做?** Mamba 社区与目标检测社区的交叉仍处于早期(Mamba-YOLO 刚起步); "频域+SSM"跨两个小众方向的组合门槛高; "修复"比"增强"更难(formulate 和评估都更复杂)。

4. **如果我是作者,下一篇做什么?** 将 FM-CHFEM 三步流水线转化为全 CNN 架构: FFT/DCT→可学习全局相似度矩阵(大核conv/self-attention)→deformable conv 修复——这是 #11 v2.0 的最高优先级探索方向。

5. **新 Gap**: 频域判据的双重角色(门控+修复)——当前 #11 仅用高频做门控，FM-CHFEM 证明高频也可指导特征修复→"判据+修复"统一框架是无人区。

#### MFF-YOLO NWD-Soft-NMS (DMBD 2024/Springer 2025)

1. **别人没解决什么?** NMS 距离度量长期锁定 IoU，NWD-Soft-NMS 证明 NWD 对小目标+密集场景更友好(+9.0%联合增益)。但 NWD 计算开销高于 IoU; σ 参数全局固定→无密度自适应。

2. **瓶颈**: DMBD 会议论文篇幅受限→架构细节和消融不充分; 仅在 YOLOv8 验证; +9.0% 是 MFF+NWD-Soft-NMS 联合增益(NWD-Soft-NMS 单独贡献未知)。

3. **为什么没人做?** NWD(2021)在检测社区传播慢，可能是因为"Wasserstein 距离"的数学门槛; NMS 改进通常被视为"低创新性工程优化"，顶会不友好。

4. **如果我是作者,下一篇做什么?** (a) 密度自适应 σ: 局部密度高→σ↑(更宽容); (b) 频域距离度量替代 NWD→可能对遮挡更鲁棒。

5. **新 Gap**: NMS 与特征级判据的协同——频域判据在特征层标识密集区域+NWD-Soft-NMS 在后处理层保留密集框→完整的"密集友好"检测管线，无人端到端设计。

#### GCS-DETR (Multimedia Systems 2026) 🟪 YOLO迁移过滤器已应用

1. **别人没解决什么?** RT-DETR backbone/Neck 未针对小目标高频细节保留和遮挡场景做专门优化——GCS-DETR 的 FreqDyNet/OAM-FPN/NWD-MPDIoU 三个组件均填补此空白，且**全部架构无关可迁移 YOLO**。

2. **瓶颈**: "动态"机制未确认(输入自适应 vs 学习参数固定)→若是后者则落入频域增强范式(非条件计算); 三组件消融贡献未知; +3.0-3.5% 在密集遮挡文献中偏低。

3. **为什么没人做?** 频域滤波在 DETR backbone 中的应用仍属少数(主流用标准 ResNet/Swin); 小波变换在 Neck 中的应用更是罕见。

4. **如果我是作者,下一篇做什么?** (a) NWD-MPDIoU 在 YOLOv11 上的架构无关性验证; (b) OAM-FPN 小波高频保留→YOLO PANet 嵌入。

5. **新 Gap**: 连续频域滤波(学习权重) vs 离散频域门控(硬阈值)的 Pareto 前沿——FreqDyNet 连续软权重(无信息丢失但不省算力) vs #11 离散硬门控(有信息丢失风险但真省算力)→最优折中方案未探索。

#### Repulsion Loss (CVPR 2018) — 经典补读 L2

1. **别人没解决什么?** RepLoss 用排斥损失解决密集遮挡（训练期推开预测框），但遮挡感知是被动的（仅靠 IoU 阈值判断哪些 GT 需要排斥），无显式遮挡先验。三个超参数(α/β/σ)全局固定，无场景自适应。

2. **瓶颈**: IoG 对极度重叠(IoG→1)的惩罚饱和（Smooth_ln 截断）；RepBox O(N²)在密集 anchor 一阶段检测器中计算爆炸；仅作用回归分支，分类置信度与定位质量的 misalignment 没有缓解。

3. **为什么没人做?** RepLoss 提出后社区注意力被更"现代"的方案（如 DETR 集合预测、显式遮挡图）吸引。但 RepLoss 的核心洞察 — "训练期排斥 = NMS 问题前置解决" — 仍有未被充分挖掘的价值，尤其与频域判据结合。

4. **如果我是作者,下一篇做什么?** (a) 频域自适应 α/β:遮挡区 α↑(强排斥)/非遮挡区 α↓; (b) RepBox 高效化:KNN 近似或空间哈希; (c) 分类-回归一致性:IoG 排斥信号反馈分类分支。

5. **新 Gap**: 频域判据作为 RepLoss 权重调节器的上游输入 — 高频异常→遮挡→加重排斥权重 → 免标注的遮挡感知 RepLoss。这一「频域 + 排斥损失」的组合至今无人探索。

#### CrowdDet (CVPR 2020 Oral) — 经典补读 L2

1. **别人没解决什么?** EMD Loss 证明"一个 proposal 预测 K 个实例"可以有效解决密集重叠问题。但 K 值是静态的(K=2)，密集区浪费/极密集区不足；方法锁定在 proposal-based 检测器，YOLO 一阶段无直接迁移路径。

2. **瓶颈**: O(K!)匹配复杂度限制 K 不可扩展(K>3)；预测实例间无硬互斥约束(EMD 匹配隐含抑制但可收敛到同一实例)；依赖 proposal 质量(proposal 偏差→G(b_i)=∅)；Set NMS 仅对 proposal-based 有效。

3. **为什么没人做?** CVPR 2020 Oral 后社区注意力转向 DETR(Hungarian 全局集合预测)和 Transformer。CrowdDet 的「局部集合预测」概念在两阶段检测器社区被视为"已解决"，但其对 YOLO 的潜在迁移价值被完全忽略。

4. **如果我是作者,下一篇做什么?** (a) 自适应 K:频域/密度判据驱动 per-proposal K 值; (b) YOLO grid-cell EMD:将 EMD 从 proposal 级搬到 P2 grid cell 级; (c) EMD + RepLoss 联合:proposal 内匹配 + proposal 间排斥。

5. **新 Gap**: 「局部集合预测」(CrowdDet) + 「全局集合预测」(DETR) 的混合架构 — YOLO 的每个 grid cell 独立做局部 EMD 匹配 + 全局匈牙利匹配处理跨 cell 重复 — 两阶段集合预测的层次化设计。这一 Gap 位于 YOLO 和 DETR 两条路线的交叉点，是双轨融合的潜在突破口。

#### Soft-NMS (ICCV 2017) — 经典补读 L2

1. **别人没解决什么?** Soft-NMS 用分数衰减替代硬清零 — 一行代码改善 NMS 的误杀问题。但距离度量仅用 IoU（对小目标不敏感），衰减函数手工固定（不可学习），σ 全局静态（无密度自适应）。

2. **瓶颈**: 贪心算法的根本局限未解决（最高分框定位不准→衰减基准错误）；IoU 度量无法区分"同实例多预测"vs"不同实例重叠"；无类别感知；无内容感知。

3. **为什么没人做?** NMS 改进被社区视为"低创新性工程优化"（顶会不友好）。NWD-Soft-NMS(MFF-YOLO 2025)已沿"改进距离度量"方向前进，但"可学习衰减函数"和"内容感知衰减"两个更根本的方向仍空白。

4. **如果我是作者,下一篇做什么?** (a) 密度自适应 σ:密集区大 σ/稀疏区小 σ; (b) 频域感知衰减:IoU/NWD+框内频谱相似度双因子; (c) MLP 可学习衰减函数。

5. **新 Gap**: NMS 的「内容感知」— 当前所有 NMS 变体仅用 box 坐标(空间重叠)判断冗余，忽略框内特征内容。两个不同实例即使 90% IoU，其框内纹理/边缘/频谱模式可能完全不同 — 内容感知 NMS 可保留这些框。频域判据(#11)天然适合度量内容相似性(频谱相似度→独立于空间重叠的第二判断轴)。「频谱感知 Soft-NMS」是完全未被探索的方向。

## 🔴 交叉分析: 密集遮挡 × 已有研究方向（2026-07-18 TASKS 交叉分析）

> K1 Gap 分析完成后，按 TASKS 要求执行三项交叉分析。目的：确定密集遮挡方向与项目主线的交叉点、张力点和协同机会。

### 交叉一：密集遮挡 × P2 稀疏化（#5 熵判据在密集场景是否失效？）

**核心张力**: #5 用语义熵识别背景→跳过计算。但密集遮挡区域也可能高熵（多个目标特征混合→语义不纯→高熵）。如果 #5 把密集遮挡区误判为"背景"→跳过计算→密集场景精度崩塌。

**张力分解——密集区高熵的可能成因**:

| 高熵成因 | 机制 | 是否应跳过？ | 应对 |
|---------|------|------------|------|
| 背景冗余（空旷路面/天空） | 无目标→特征均匀→高熵 | ✅ 应跳过（#5 正确） | #5 本体 |
| 特征弱（遮挡导致信号衰减） | 目标被遮挡→CNN 特征弱→高熵 | ❌ 不应跳过→应增强（OPL 立场） | 频域判据检测遮挡→降稀疏率 |
| 语义混合（多目标邻域重叠） | 不同目标特征在邻域叠加→高熵 | ❌ 不应跳过→密集区需全算 | 密度判据检测密集→降稀疏率 |
| 纹理复杂（建筑物/树等背景纹理） | 非目标的复杂纹理→高熵 | ✅ 应跳过（非目标） | #5 语义判据可区分 |

**关键洞察**: #5 当前的"高熵=跳过"假设将所有高熵归因为"背景冗余"——这在稀疏场景成立，但在密集遮挡场景可能误杀后两种（特征弱/语义混合）。**#5 需要一个"遮挡/密集感知"的稀疏率修正模块**。

**解决方案——双判据门控**:
1. **主判据（语义熵）**: 识别候选跳过区域（高熵→候选跳过）
2. **否决判据（频域密度+遮挡先验）**: 在高熵候选区中，频域高频异常度↑或局部密度↑→否决跳过→保留全计算
3. **效果**: 稀疏场景（低密度+低遮挡）→高稀疏率；密集场景（高密度或高遮挡）→低稀疏率→自动调节

**结论**: #5 熵判据在密集场景**不会系统性失效**，但需要增加"密集/遮挡否决"机制——频域密度+遮挡先验作为第二判据。#31（密度×频域联合判据）提供了这个否决机制的实现框架。

**关联 Idea**: #5 v3.2（遮挡感知稀疏率修正）、#31（密度×频域双维条件计算）、#36（遮挡区高熵成因裁决）

### 交叉二：密集遮挡 × DETR（query 去重与端到端检测）

**背景**: DETR 的 Hungarian 匹配理论上可消除 NMS（全局二分匹配→每 GT 唯一 query），但在密集遮挡场景中：
- Dome-DETR 仍保留 Dynamic NMS（端到端承诺破防）
- D³R-DETR 也使用 NMS 后处理
- 密集场景的 query 去重是连 DETR 都没解决的问题

**YOLO 视角的独特优势**: YOLO 的 grid-cell 结构天然支持空间局部性——密集区域自然获得更多 grid-cell→更多预测→不需要复杂的 query 分配。但缺少 DETR 的全局去重机制。

**DETR→YOLO 可迁移概念**:

| DETR 概念 | YOLO 等价 | 迁移难度 | 关联 Idea |
|-----------|----------|---------|----------|
| Hungarian 二分匹配 | Grid-cell EMD 局部匹配 | ⚠️ 中 | #37 |
| Query 去重（自注意力） | Set NMS 同源跳过 | ✅ 低 | #37 附 |
| PAQI 密度自适应 query | 连续密度 LA K(ρ) | ✅ 低 | #40 |
| MWAS 浅层 token 稀疏 | #5 P2 空间门控 | ✅ 低（已有） | #5 |
| RCDR 中心正则（判据无关） | 同款 L2 正则 | ✅ 低（零成本） | #30/#5 可引入 |

**关键洞察**: YOLO 在密集场景的"去重"问题比 DETR 更容易解决——因为 grid-cell 的空间局部性天然限制了重复预测的范围（同 cell 或邻 cell），而 DETR 的全局 query 可以在图像任意位置产生重复。CrowdDet EMD 的局部集合预测 + Set NMS 是比 Hungarian 匹配更契合 YOLO grid 结构的方案。

**结论**: 密集遮挡方向的 DETR query 冲突问题对 YOLO 迁移价值**中等**——概念可借鉴（密度自适应预测数、局部集合匹配）但实现路径完全不同。YOLO 的密集去重应沿 #37（Grid-Cell EMD）+ #40（连续密度 LA）路线，而非照搬 DETR Hungarian。

**关联 Idea**: #37（YOLO Grid-Cell EMD）、#40（连续密度 LA）、#39（EMD+RepLoss 联合——训练期匹配+排斥覆盖 YOLO 的去重需求）

### 交叉三：密集遮挡 × 频域（密集边缘 vs 孤立目标频谱差异）

**核心问题**: 频域判据能否区分"密集目标群的边缘"和"孤立目标的边缘"？如果能→频域判据的价值从"通用前景判据"升级为"场景自适应判据"。

**频谱差异假说**:

| 场景 | 空间特征 | 频谱特征 | 对频域判据的影响 |
|------|---------|---------|----------------|
| 孤立小目标 | 单一边界、点状分布 | 低频→中频集中、高频点状孤立 | 高频局部异常度↑→目标指示可靠 |
| 密集目标群 | 多边界交织、区域分布 | 高频密集分布（多边缘叠加）、频谱复杂度高 | 高频异常度仍↑但**频谱复杂度更高**→可区分 |
| 背景纹理（路面/建筑） | 规则纹理、大面积均匀 | 高频规则分布（重复纹理） | 高频异常度↓（规则→局部对比度低） |
| 遮挡边界 | 突变边界、半透明重叠 | 高频突变（多方向边缘叠加） | 高频异常度极高→遮挡指示器 |

**可区分性验证路径**:
1. **频谱复杂度度量**: 局部 patch 的频谱熵（频谱分布的 Shannon 熵）→密集区频谱熵高（多方向边缘→多频率分量均匀分布）、孤立区频谱熵低（单一方向边缘→频率集中）
2. **高频空间聚集度**: 高频响应的空间分布——孤立目标→高频点状散布、密集群→高频区域聚集
3. **联合判据**: 高频能量（强度）× 频谱熵（复杂度）× 空间聚集度（分布模式）→三维频域判据→区分孤立/密集/遮挡/背景

**对 #11/#30 的直接影响**:
- **当前判据**: 高频能量（一维）→"哪里有边缘/纹理"
- **升级判据**: 高频能量 + 频谱复杂度 + 空间分布 → "这里是什么样的场景"→自适应后续处理
- **应用**: 密集区→降稀疏率(#5)/调大 NMS σ(#41)/O2O LA(#40)；孤立区→标准处理；遮挡区→增强特征恢复(#35)

**结论**: 密集边缘 vs 孤立目标的频谱差异**在理论上是可区分的**（频谱复杂度不同），但需要 VisDrone 真实数据的统计分析验证（AUROC of spectral complexity vs. GT density classification）。这是 #25（频率签名判据）在密集遮挡方向的自然延伸——从"点/线/面"频率签名升级为"孤立/密集/遮挡"场景签名。

**关联 Idea**: #25（频率签名判据→场景签名扩展）、#35（频域遮挡先验→频谱复杂度用于遮挡检测）、#31（密度×频域联合→频谱复杂度作为密度正交维）

### 来自 FAA (CVPR 2026) —— 🟠🔬 深读 2026-07-18, 频域×OBB首次交叉

### 来自 DERNet (arXiv 2026.06) —— 🟡🔬 P0 深读 2026-07-19, 频域全管线特征学习最系统化方案

1. **别人没解决什么?** ①DERNet将频域处理做到全管线(backbone WDG+neck LGE+head FDHead)但全部停留在"增强"维度——WDG的HF门控g已计算per-location高频重要性，但仅用于特征增强而非计算分配; ②FDHead占DERNet-S总GFLOPs 55.6%(7.40/13.3)→频域增强的算力代价被暴露但未被解决; ③三模块联合(0.458)反而略低于WDG+FDHead(0.464)→Neck-Head频域过度增强的问题未被诊断; ④WDG的"HF as gate"哲学与#5语义熵门控功能同构(都是per-location前景/背景信号)但信息源完全不同(频域小波vs语义)→双源门控对比空白。

2. **瓶颈**:
   - 频域增强的算力代价:FDHead 55.6% GFLOPs→与"参数效率极高(1.3M)"的叙事矛盾——真正轻量的是backbone/neck，head反而很重
   - 三模块互补性:WDG+LGE+FDHead不如WDG+FDHead→LGE在WDG+FDHead基线上的增量价值存疑
   - 绝对精度提升有限:DERNet-S仅+0.005 mAP50 vs YOLOv11-S→核心优势是参数效率而非精度突破
   - VisDrone绝对性能不高:0.316(DERNet-S)/0.362(DERNet-M)→与PRNet 54.1/HEdge-MamYOLO 52.5差距大(但DERNet用1.3M vs PRNet 24.6M)
   - 非严格同构:C3_Faster+通道缩减+P2 head叠加→"1/7参数"不全是频域模块的贡献

3. **为什么没人做?** 频域+检测社区仍在探索"怎么用频域提升特征质量"(2025-2026累计≥7篇全增强范式)。条件计算/稀疏化需要"质量vs效率"的思维转换——社区惯性:既然频域能提升特征→自然做增强; 从增强到节省需要认识到"不是所有位置都需要频域增强→高频区域需要增强,低频区域可以跳过"。

4. **如果我是作者,下一篇做什么?**
   - 频域引导的条件计算:WDG的HF门控g→P2像素级计算决策(高g→保留+频域增强; 低g→跳过+标准卷积)→同时实现"高g区增强"和"低g区节省"
   - FDHead轻量化:SHG的Haar DWT+Gate→仅用WDG的g信号引导box分支注意力→省7.40 GFLOPs
   - 频域增强×遮挡交叉:遮挡区域高频被破坏→门控应降低而非增强→"遮挡感知频域门控"

5. **新Gap: 频域判据双用途——计算分配+特征增强的统一框架空白**——WDG的g信号已证明可做特征增强(DERNet)，理论上也可做计算分配(#11)→但无人探索"同一频域判据同时服务增强和节省": 高g→全保留+频域增强; 中g→保留+标准卷积; 低g→跳过——三重决策的频域统一框架是全新概念空间。

6. **新Gap: DERNet vs #11 的"增强vs节省"路线实验裁决**——两篇论文都基于"频域对微小目标很重要"的前提，但走了相反的技术路线(DERNet增强/#11节省)。同一个YOLOv11-S+VisDrone基准上的头对头对比(精度/FLOPs/延迟/可解释性)是对频域方向的根本性裁决，且从未被做过。

7. **新Gap: WDG门控→#5语义熵门控的双源对照**——WDG(HF小波系数)和#5(CLIP语义熵)都产生per-location的[0,1]前景信号→信息源完全正交(频域纹理vs语义空间)。"频域门控vs语义门控"的判据融合实验不存在→融合是否互补(频域边界信息+语义内容信息=更准确的前景判别)是待验证假设。

8. **新Gap: DERNet频域管线×YOLO-Master MoE路由的交叉空白**——DERNet LGE的方向响应(K=2方向)可作为YOLO-Master ES-MoE的路由信号→方向特征决定选择哪个专家的卷积核方向→"频域引导的感受野选择"是全新交叉维度。

9. **新Gap: LGE方向响应→OBB方向先验迁移**——LGE的K个方向响应强度与FAA FAE的FFT角度估计共享"从频域读方向"的哲学——LGE的方向响应可直接作为OBB检测的方向分类辅助特征(零额外频域变换,复用Neck已有计算)→"小波方向门控→旋转框方向估计"是频域×OBB的新交叉点(RDCNet极坐标DCN的互补路线)。

10. **关联 Idea**: #11(频域→条件计算·增强vs节省路线分化) / #5(WDG门控vs语义熵双源对照·R2) / #30(频域判据跨架构通用·WDG g→DETR token重要性) / D1(频域三线共享·增强维度补充) / OBB×频域交叉(LGE方向→旋转框)

### 来自 VALA (Neurocomputing 2026) —— 🟡🔬 P0 深读 2026-07-19, 首次将锚框尺度引入标签分配

1. **别人没解决什么?** ①此前LA方法要么改"如何选正样本"的规则(ATSS统计阈值/SimOTA cost matrix)，要么改"用什么度量相似度"(RFLA RFD/NWD/DotD)——从未有人改"参与度量的锚框本身"的尺度; ②anchor-object尺度失配是单独问题维度→此前被密度(DALA)/任务对齐(TAL)/几何感知(RFLA)维度掩盖; ③IoU一致性(train-eval metric alignment)是RFLA/NWD/DotD等替换度量方法的系统性缺陷→VALA通过"改锚框不改IoU"首次同时解决尺度失配和度量一致性问题。

2. **瓶颈**:
   - 仅anchor-based验证→YOLO anchor-free迁移需要概念重定义(虚拟anchor point/逐层scale range重校准)
   - VisDrone绝对值受baseline检测器制约(29.4 AP·无架构改进)
   - 虚拟锚框尺度为静态统计(训前固定)→无法适应域偏移和未知目标尺度
   - DSS schedule为固定epoch-driven→per-dataset调参负担
   - 论文无arXiv+付费墙+代码未发布→复现风险最高

3. **为什么没人做?** 标签分配社区的研究焦点一直在"选择规则"和"度量函数"两个维度——锚框尺度被视为检测器架构的一部分（预设超参）而非LA的创新维度。VALA首次将其从"预设超参"升级为"数据驱动的可校准参数"→开辟了LA的第三创新维度（规则/度量/尺度）。

4. **如果我是作者,下一篇做什么?**
   - Anchor-free适配: 逐层scale range统计重校准替代虚拟锚框→直接应用于YOLO SimOTA/TAL
   - 尺度×密度双维LA: VIoU+DALA联合→同时控制"锚框尺度"和"正样本数"
   - 频域判据→动态虚拟锚框: 高频能量强度实时调整锚框尺度(高频强→锚框可稍大→条件放宽)

5. **新Gap: YOLO anchor-free的"虚拟锚框"等效设计空白**——YOLO使用anchor-free检测头(grid cell中心点作为锚点)→VALA的虚拟锚框概念需要等效实现: ①逐层scale range重校准(根据YOLO每层负责的目标尺寸统计→调整SimOTA/TAL中每层的scale range) ②虚拟anchor point偏移(根据目标尺寸分布→偏移anchor point位置→优化微小目标的特征图位置对齐)。两种方案均未被探索。

6. **新Gap: 尺度×密度双维LA——#40的尺度维扩展**——#40连续密度LA控制"每GT多少个正样本"(数量维), VALA控制"锚框多大尺度"(尺度维)→二者完全互补且不冲突。联合设计=标签分配的完整二维控制: 密集+微小=多正样本+小锚框; 稀疏+大目标=少正样本+大锚框。这是#40最明确的技术升级方向。

7. **新Gap: 频域判据→动态虚拟锚框——统计→动态的范式升级**——VALA虚拟锚框尺度基于训练集静态统计(训前固定)→频域判据(高频能量强度)可实时反映per-instance的边缘质量→高频强→目标边缘清晰→锚框可稍大(正样本条件放宽); 高频弱→目标模糊→锚框需精确匹配(正样本条件收紧)。统计→动态的升级使VALA从"数据驱动校准"升级为"输入自适应校准"。

8. **新Gap: VALA→OBB旋转框的虚拟锚框校准**——OBB检测中旋转框的尺度变化极端(长边可达短边5-10×)→VIoU在OBB场景的收益可能更大。但虚拟锚框的旋转角度如何校准? 按方向分布统计做per-angle-bin虚拟锚框? 或与FAA FAE的频域方向估计联合→方向感知虚拟锚框?

9. **关联 Idea**: #40(连续密度LA·尺度维扩展·双维LA) / #12(KLD分配·度量维对照) / #11(频域判据→动态虚拟锚框) / OBB×尺度交叉(旋转框VIoU)

## 🟡 K1 知识补充：尺度变化方向系统化知识体系（2026-07-19）

> 基于 YOLO-Master(CVPR 2026) / DERNet(arXiv 2026) / VALA(Neurocomputing 2026) / FS-Mamba(Displays 2026) 四篇 P0 深读 + 尺度 L1 检索(30篇) → 三 KB 文件(loss/augmentation/head)尺度专节补充完成。以下从跨论文交叉视角识别系统性 Gap。

### K1-1. 标签分配的三维创新空间——双维及以上无人涉足

**现状**: LA存在三个独立创新维度——规则维(ATSS→SimOTA→DALA·控制"选几个正样本")、度量维(IoU→KLD→NWD·控制"用什么度量匹配质量")、尺度维(VALA·控制"锚框应该多大")。

**Gap**: 
- DALA仅覆盖规则维(密度→O2O/Decreasing)，VALA仅覆盖尺度维(虚拟锚框→尺度重校准)→**密度×尺度双维联合LA完全空白**
- 三维(规则×度量×尺度)自适应LA的理论最优解尚未被探索
- 所有LA方法中仅DALA考虑目标空间分布(密度)，其他方法假设目标独立均匀分布→**空间上下文感知的LA是更上层的缺失维度**

**项目切入**: #40(连续密度感知LA)可从密度维扩展到密度×尺度双维——每个GT的K个正样本中，每个样本的匹配质量同时考虑local density(几个GT竞争)和scale match(锚框大小是否合适)→联合密度-尺度软标签分配

### K1-2. Anchor-free检测器的尺度感知LA适配——跨架构迁移方法论空白

**现状**: VALA、DCNet、SA-Matching DETR等尺度LA方法均为anchor-based设计(依赖显式锚框进行尺度重校准)→YOLO anchor-free(每grid cell直接回归)无显式锚框。

**Gap**: 
- "anchor-free等效尺度校准"的方法论尚未建立——RFLA的ERF建模提供了一条路径(感受野=隐式锚框)但未与尺度校准结合
- YOLO26 STAL(小目标膨胀代理框)是anchor-free尺度感知的唯一实践，但仅影响mask不影响回归→是部分解决方案
- Anchor-free场景下"逐层scale range"应如何定义？grid cell感受野 vs 锚框尺度的对应关系需要形式化

**项目切入**: #6 baseline(YOLO11+STAL)是anchor-free尺度感知的起点→升级路径: STAL(候选膨胀) + VALA anchor-free等效(感受野缩放) + DALA(密度自适应K)

### K1-3. 增强vs节省路线——频域特征学习的根本性方法论选择

**现状**: DERNet=频域→特征增强(所有位置做频域变换, 1.3M参数/13.3GFLOPs→VisDrone 0.316) vs #11=频域→条件计算(高频区算·低频区跳过, 目标省算力)。DERNet FDHead 55.6% GFLOPs占比=增强路线的算力代价定量暴露。

**Gap**: 
- "增强vs节省"在同一YOLO+VisDrone上的实验性对照从未被做过→频域在检测中的ROI(投入产出比)根本性方法论问题
- 两条路线可能的最优方案是混合型: 用增强范式处理所有位置(保精度)·但用条件计算范式分配算力(控FLOPs)→"增强且稀疏"的双模设计是空白
- WDG的"HF as gate"(自派生门控)可以直接作为#11判据的输入→同架构内同时支持增强和节省(开源双mode)是新颖架构设计

**项目切入**: #11的差异化定位就是"节省"路线→DERNet增强路线的定量化算力代价(=55.6% FLOPs)是其最强动机数据点

### K1-4. 训练-推理解耦的三范式系统对比——方法论指导空白

**现状**: 三种训练专用·推理零开销范式独立发展: ①辅助Loss训后丢弃(SET CVPR 2025·频谱增强教师→蒸馏) ②辅助Head训后丢弃(FS-Mamba Displays 2026·SR重建头→训后移除) ③门控训后硬化(#5 Gumbel-Softmax→Hard阈值)。

**Gap**:
- 三者在同一YOLO+VisDrone基准上的精度/稳定性/超参敏感性/训练墙钟/收敛速度的系统对比完全不存在
- 每种范式的适用场景不同: ①适合有教师模型时(需额外训练); ②适合特征增强(架构无关·最简单); ③适合门控/稀疏化(需Gumbel退火). 但何时用哪种→无指南
- 三种范式可能可叠加(如FS-Mamba SR辅助头 + #5 Gumbel门控)→叠加后的干扰/协同效应未知

**项目切入**: "训练-推理解耦的设计空间与选择指南"可作为独立分析性论文——三范式在统一YOLO+VisDrone上的系统对比+叠加实验→对社区有方法论文档价值

### K1-5. 检测头的条件计算——Backbone繁荣 vs Head荒芜的结构性不对称

**现状**: Backbone已有丰富的条件计算机制(YOLO-Master ES-MoE感受野路由/#5空间门控/MGS通道门控/HI-MoE专家路由)，而检测头几乎全部是固定结构——YOLO Head = 固定双分支(分类+回归) × 固定三层(P3-P5)。

**Gap**:
- DERNet FDHead的SHG是Head端**唯一**的空间自适应信号——且仅做权重调整(高频→高权重)，不涉及计算跳过
- "Head的条件计算"是完全未探索的方向: 不同位置的Head预测可不同精度(密集区全精度·稀疏区轻量Head)/不同层级的Head可不同激活(大目标只用P5 Head)
- YOLO-Master发现Backbone-only路由最优·级联梯度冲突——Head端路由是否也有类似冲突? 还是Head路由独立于Backbone路由?

**项目切入**: #5目前限定P2 Backbone端→扩展到Head端(P2 Head也做条件计算)=Head端条件计算蓝海探索

### K1-6. 尺度自适应数据增强——SR辅助 vs 动态分辨率 vs 频域分解

**现状**: 三种特征级/数据级尺度增强路线: ①SR辅助训练(FS-Mamba·训练期超分→特征增强·零推理) ②动态分辨率上采样(DARE-YOLO DRDU·内容自适应·推理期) ③频域多尺度分解(DERNet WDG·Haar DWT·推理期)。

**Gap**:
- 三种路线的增益叠加性未验证——SR辅助训练+动态分辨率+频域分解同时使用→增益相加还是边际递减?
- SR辅助训练的最优目标函数(SR重建图像 vs SR重建边缘 vs SR重建高频残差)从未被消融
- 频域分解的"最优基函数"(Haar vs Daubechies vs Log-Gabor vs 学习式)在检测任务上无系统对比

### K1-7. 尺度×密度联合建模——两个正交维度的融合设计空间

**现状**: 尺度变化和密集遮挡在VisDrone场景高度相关(拥挤区=小目标群)→但项目的两个方向(🟡尺度+🔴密集)是分开研究的。

**Gap**:
- 尺度-密度联合分布建模: VisDrone中"小且密集"区域 vs "大且稀疏"区域的分布特征→可作为条件计算的第三维路由信号
- DALA(密度感知LA) + VALA(尺度感知LA)的双维联合→DALA的"密集区O2O vs 稀疏区Decreasing"+VALA的"小目标大锚框 vs 大目标小锚框"→联合决策表(4种组合×4种策略)
- 频域判据可同时服务尺度和密度: 高频能量=小目标聚集区(尺度指示)+高频局部异常度=遮挡边界(密度指示)→同一个判据图双用途

### 一、跨P0四篇交叉空白（Systematic Cross-P0 Gaps）

> 四篇P0各自解决尺度变化的一个子问题：YOLO-Master=感受野维条件计算、DERNet=频域特征增强、VALA=尺度感知标签分配、FS-Mamba=频域门控+SR辅助训练。交叉空白 = 四篇方法的交集处未被探索的空间。

#### G1-S1 🔴 频域判据→MoE路由信号：条件计算双维融合空白
- **涉及论文**: YOLO-Master × DERNet × FS-Mamba
- **描述**: YOLO-Master ES-MoE的GAP路由是图像级空域统计（`GAP→γ=8→Soft→Hard Top-K`），完全丢失空间粒度。DERNet WDG的HF自派生门控`g∈(0,1)`和FS-Mamba FDGate都是**逐位置**的频域响应——天然适配空间级MoE路由。**但频域判据从未被用作MoE路由信号**。
- **为什么是Gap**: 
  - GAP→图像级路由 = 全图所有位置路由到同一组专家 → 密集区/稀疏区/遮挡区无差别处理
  - 频域判据可提供空间级路由: 高频区→小kernel专家(细节保留)、低频区→大kernel专家(语义聚合)、遮挡区→混合专家(上下文推理)
  - YOLO-Master Wiki自认"后期版本revert to homogeneous experts"→异构专家的价值被GAP路由掩盖（路由粒度太粗导致异构专家优势无法发挥）
- **可验证性**: ⏸ 需实验（频域图→空间路由 vs GAP路由→同一MoE架构的VisDrone消融）
- **关联 Idea**: #11(频域判据·路由信号化→R1扩展)、#5(空间级门控与MoE路由融合)

#### G1-S2 🔴 "增强+节省"统一频域框架——双模频域处理无人区
- **涉及论文**: DERNet × #11
- **描述**: DERNet = 频域→增强（所有位置做频域变换提升特征质量），#11 = 频域→节省（高频区算·低频区跳过）。两条路线在同一检测器中从未共存。实际上WDG的HF门控`g`已经计算了per-location重要性——同一个`g`信号可以**同时决定增强强度（g→WDG残差权重）和计算分配（g→是否激活FDHead）**。
- **为什么是Gap**: 
  - 当前路线选择是"全选增强"(DERNet)或"全选节省"(#11规划)——但自然方案是"高频区增强+全算、中频区标准卷积、低频区跳过"
  - 三重决策（增强/标准/跳过）同一门控信号→一次计算三次收益
  - DERNet FDHead 55.6% GFLOPs = 增强的定量代价→若能"增强高g区 + 跳过低g区"可同时超越纯增强和纯节省
- **可验证性**: ⏸ 需实验（同一YOLO+VisDrone上DERNet增强模式 vs #11节省模式 vs 混合模式三方头对头）
- **关联 Idea**: #11 v2.0(双模频域·增强+节省统一框架)

#### G1-S3 🟠 尺度×密度双维标签分配——LA第三维度的理论最优解
- **涉及论文**: VALA × DALA
- **描述**: DALA控制"正样本数量"（密度维·O2O vs Decreasing LA），VALA控制"锚框尺度"（尺度维·虚拟锚框重校准）。两者完全独立互补→联合 = 标签分配的完整二维控制。**但双维联合的联合决策表（4种密度×尺度组合→4种LA策略）从未被设计。**
- **为什么是Gap**: 
  - 联合决策逻辑: 密集+微小→O2O+小虚拟锚框（精准匹配·避免重复）+多正样本；稀疏+大目标→Decreasing LA+标准锚框+少正样本
  - 两者的衰减schedule（DALA K(t) 递减 + VALA DSS渐进衰减）若不同步→训练中可能产生矛盾信号
  - 频域判据可提供密度+尺度的统一上游信号（高频能量→同时指示密度和尺度）
- **可验证性**: ⏸ 需实验（DALA + VALA的联合消融·4种组合策略在VisDrone上的对比）
- **关联 Idea**: #40(连续密度LA·尺度维扩展→双维LA)·这是#40最明确的升级方向

#### G1-S4 🔴 训练-推理解耦三范式系统对比——方法论级空白
- **涉及论文**: FS-Mamba × SET × #5
- **描述**: 三种"训练专用·推理零开销"范式独立发展: ①辅助Loss训后丢弃(SET CVPR 2025·频谱增强教师) ②辅助Head训后丢弃(FS-Mamba SR头) ③门控训后硬化(#5 Gumbel→Hard阈值)。**三者在同一YOLO+VisDrone上的系统对比完全不存在。**
- **为什么是Gap**: 
  - 每种范式的适用场景不同: ①适合有教师模型时 ②适合特征增强(架构无关·最简单) ③适合门控/稀疏化(需Gumbel退火)。社区缺乏"何时用哪种"的方法论指南
  - 三种范式可能可叠加（FS-Mamba SR辅助头 + #5 Gumbel门控 + SET辅助Loss→同时存在）→叠加干扰/协同效应未知
  - 这是一个**纯分析性·零实验即可文档推演70%**的Gap→适合作为独立分析性论文
- **可验证性**: ✅ 70%可文档推演（三范式的数学形式化+适用场景分析+叠加兼容性矩阵），30%⏸需实验（三方VisDrone对照）
- **关联 Idea**: 新Idea候选——"训练-推理解耦设计空间与选择指南"（分析性论文·可文档推演为主）

#### G1-S5 🟠 频域门控——可学习(FDGate) vs 免训练(#11 S1)路线裁决空白
- **涉及论文**: FS-Mamba × #11
- **描述**: FS-Mamba FDGate = 可学习门控高通滤波（1×1 Conv+Sigmoid→gated HF injection·BCE训练），#11 S1 = 免训练空域高通代理（Sobel/LoG→高频响应统计·零参数）。两者功能等价（识别高频→优先处理），但实现路线相反。**同一YOLO+VisDrone上的head-to-head对比从未被做过。**
- **为什么是Gap**: 
  - 可学习门控的优势: 可适应数据集特定的高频模式（如VisDrone的城市纹理 vs 海面纹理）·但需GT监督
  - 免训练判据的优势: 零参数·零训练·跨数据集泛化（物理先验）·但无数据集自适应
  - 判决结果决定项目的频域门控路线选择（#11当前走免训练·若可学习显著更优则需战略调整）
- **可验证性**: ⏸ 需实验（FDGate vs #11 S1在同一YOLO+VisDrone上的精度/延迟/数据效率对比）
- **关联 Idea**: #11(若可学习更优→#11 v2.0路线修订) / #30(同一对照在DETR侧执行)

#### G1-S6 🟡 尺度自适应的空间粒度断层——图像级→层级→位置级
- **涉及论文**: YOLO-Master × VALA × DERNet
- **描述**: 四篇P0的尺度自适应粒度形成谱系——YOLO-Master GAP=图像级（全图同一路由）/ VALA s^l=层级（同层所有位置共享同一尺度因子）/ DERNet WDG g=位置级（逐位置[0,1]门控）/ FS-Mamba FDGate=位置级。**但没有任何方法同时利用多个粒度。**
- **为什么是Gap**: 
  - 多粒度联合决策: 图像级决定全局路由策略→层级决定每层的尺度因子→位置级决定该位置是否需要频域增强
  - 不同粒度携带不同信息: 图像级=场景类型(城市vs海面)、层级=目标尺度分布、位置级=局部纹理复杂度
  - 三粒度门控的协同设计空间完全未被探索
- **可验证性**: ⏸ 需实验（多粒度消融·图像级+层级 vs 图像级+位置级 vs 三层全叠加）
- **关联 Idea**: #5 v3.2(多粒度稀疏化·图像级稀疏率+位置级门控)

#### G1-S7 🟠 DERNet LGE方向响应→OBB方向先验的迁移空白
- **涉及论文**: DERNet × FAA × RDCNet
- **描述**: DERNet LGE的Log-Gabor滤波器组（K=2方向·S=1尺度）输出方向响应强度——与FAA FAE的FFT角度估计共享"从频域读方向"的哲学，但实现完全不同（小波方向响应 vs FFT极坐标角度）。**LGE的方向响应从未被用作OBB的角度估计源。**
- **为什么是Gap**: 
  - LGE已在Neck中计算（零额外频域变换）→其K个方向响应可直接作为OBB检测头的方向分类辅助特征
  - Log-Gabor方向响应（空间-频域联合）vs FAA FFT角度（纯频域）→两种方向信息源互补
  - RDCNet极坐标DCN的θ分量与此也可联合→频域方向+空间DCN方向+Log-Gabor方向→三源方向感知
- **可验证性**: ⏸ 需实验（LGE方向响应 vs GT角度的Spearman ρ相关性→确定是否可作为方向先验）
- **关联 Idea**: OBB×频域交叉(新Idea) / FAA FAE对照 / RDCNet θ→三源方向融合

#### G1-S8 🟡 四篇P0的联合优化——端到端尺度自适应检测器设计空白
- **涉及论文**: 全部四篇
- **描述**: 四篇P0各自优化检测器的一个阶段: YOLO-Master→Backbone路由 / DERNet→Backbone+Neck+Head频域增强 / VALA→训练期标签分配 / FS-Mamba→Backbone频域门控+Neck频率保持+SR辅助训练。**无人设计端到端的尺度自适应检测器（联合优化Backbone路由+Neck频域融合+Head条件计算+LA尺度校准）。**
- **为什么是Gap**: 
  - 四组件可能冲突: MoE路由的梯度与频域增强的梯度可能互相干扰（YOLO-Master已发现Backbone+Neck路由的级联梯度冲突）
  - 但"冲突"本身是工程实现问题而非理论不可能——精心设计的梯度隔离/交替训练可能解决
  - 联合框架的潜在收益巨大: 四组件的叠加增益 → 尺度自适应检测器的"终极形态"
- **可验证性**: ✅ 概念推演（联合设计文档·非实验），实验验证⏸远期
- **关联 Idea**: 远期·尺度自适应检测器统一框架（Scaling Adaptive Detector）

### 二、L1未覆盖维度（30篇检索的覆盖盲区）

> 30篇L1检索覆盖了自适应感受野/尺度感知LA/极端小目标增强/多尺度融合/频域×尺度/条件计算×尺度六大方向。以下维度在30篇中**不存在或严重不足**。

#### G1-L1 🔴 尺度感知的条件计算——输入自适应的尺度选择性计算
- **L1覆盖现状**: 条件计算×尺度仅2篇——Input-Adaptive DNN（深度+宽度双维自适应·早退机制）和YOLO-Master（感受野维）。**无一篇做"尺度感知的空间条件计算"**（根据目标尺度动态决定哪些空间位置需要计算）。
- **为什么是Gap**: 
  - 当前条件计算全部是"统一判据→统一决策"——YOLO-Master GAP路由全图统一、Input-Adaptive DNN全图统一早退深度
  - "大目标区域→P5就够了·跳过P3/P4→省算力"的层级选择、"小目标区域→P2全算·P5可跳过"的反向选择——完全无人探索
  - 与#5互补: #5是空间维门控（前景vs背景），尺度感知门控是层级维门控（P2vsP3vsP4vsP5 per-location）
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #5 v3.3(尺度感知空间门控·层级选择+空间选择双维)

#### G1-L2 🟠 频域判据→检测后处理——NMS/Soft-NMS的频域内容感知
- **L1覆盖现状**: 30篇中**零篇**将频域用于NMS或后处理。全部频域论文（8篇）=特征增强/特征融合。频域在后处理阶段的应用是完全空白。
- **为什么是Gap**: 
  - NMS仅用IoU判断冗余——两个框90%重叠但内容完全不同（不同实例）应保留
  - 频域判据天然适合内容相似度度量——框内频谱统计（高频能量分布/主方向/频谱熵）→余弦相似度→与IoU正交的内容冗余判断
  - 尺度变化使此Gap更严重: 小目标IoU对位置敏感（1px偏移→IoU暴跌）但频域特征对位置鲁棒→频域NMS对小目标特别友好
- **可验证性**: ⏸ 需实验（框内频谱相似度 vs 是否同实例GT→AUROC·类似#30 E3模式）
- **关联 Idea**: #38(频谱感知Soft-NMS——**此Gap的直接回应**)

#### G1-L3 🟠 训练期+推理期尺度增强的联合设计
- **L1覆盖现状**: SR辅助训练（FS-Mamba/MambaIR-YOLO）=纯训练期增强（零推理开销）。动态分辨率（DARE-YOLO）=纯推理期增强（DRDU双模上采样）。**无一篇联合设计训练期和推理期尺度增强。**
- **为什么是Gap**: 
  - 训练期SR辅助 + 推理期动态分辨率 → 可能的协同: SR辅助训练让Backbone学会"超分时的细节重建"→推理期动态分辨率上采样时Backbone已具备更好的高频细节提取能力
  - 也可能冲突: SR辅助训练过拟合特定上采样模式→推理期换用不同上采样方式→特征分布偏移
  - 联合设计可产生"训练期SR增强→推理期免上采样（Backbone已足够强）"的最优雅方案
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #5/#11(门控也可与训练期增强联合)

#### G1-L4 🟡 动态感受野的自适应选择判据——超越可学习路由
- **L1覆盖现状**: 自适应感受野7篇（SMBlock/GMSPPF/SAC/LSK/动态深度可分离卷积/HAT等）全部使用**可学习参数**（卷积/注意力）动态调整感受野。**零篇使用免训练物理判据（频域/熵/梯度）决定感受野大小。**
- **为什么是Gap**: 
  - 物理判据驱动的感受野选择: 高频局部异常度↑→小kernel（保留细节）；低频均匀区→大kernel（聚合上下文）
  - 频域判据与感受野的自然对应关系: 高频=精细结构=小感受野，低频=全局结构=大感受野
  - 免训练判据可让动态感受野模块零参数→比YOLO-Master ES-MoE（需路由网络+多套专家权重）轻量1-2个数量级
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #11(高频判据扩展·感受野选择) / 新Idea(频域驱动动态感受野)

#### G1-L5 🟡 尺度感知的数据增强——自适应增强强度
- **L1覆盖现状**: 多尺度增强（Mosaic/Multi-scale training/Copy-Paste）的强度是**全局固定**的——所有图像/所有区域使用相同的增强策略。小目标可能需要更强的增强（更多copy-paste/更大尺度抖动），大目标可能过增强→精度反而下降。
- **为什么是Gap**: 
  - 尺度自适应增强: 小目标密集图→增强强度↑（更多copy-paste）；大目标为主图→增强强度↓（标准方案）
  - 区域级自适应增强: 图像中不同区域的增强策略不同——小目标密集区更多copy-paste扩充、背景区无需增强
  - 频域判据可提供"哪里需要增强"的空间指示
- **可验证性**: ✅ 概念推演·⏸实验验证
- **关联 Idea**: augmentation.md已记录DCC尺度扩展→升级为区域自适应增强

#### G1-L6 🟢 尺度变化的理论建模——从经验→理论
- **L1覆盖现状**: 所有30篇都是**工程驱动**（改模块→涨点）。无一篇从理论层面建模"尺度变化如何影响检测性能"——如感受野-目标尺度的信息论关系、多尺度特征的冗余-互补理论。
- **为什么是Gap**: 
  - #23 SNR退化假设(小目标信号在深层衰减)是项目少数从理论视角切入的工作→但未扩展到尺度变化的全貌
  - 可能的理论框架: 尺度-感受野匹配的信息瓶颈、多尺度特征的互信息分解、尺度自适应的率失真理论
  - 理论工作发表难度高但引用价值大→适合作为长期锚点论文
- **可验证性**: ✅ 纯理论推演
- **关联 Idea**: #23(SNR退化→尺度变化理论扩展) / #24(信息瓶颈→尺度维信息流)

### 三、交叉维度Gap（Scale × 其他三维度）

> 尺度变化(🟡)与项目其他三个维度(🟦条件计算/🟠OBB/🔴密集遮挡)的交叉空白。

#### G1-X1 🔴 尺度×条件计算：层级条件计算——P2/P3/P4/P5的动态选择
- **涉及维度**: 🟡尺度 × 🟦条件计算
- **描述**: #5当前仅做P2层的空间门控（P2内"哪些位置算"），但尺度感知的条件计算还有一个更基础的维度——**"哪些层级算"**。大目标可能只需要P4/P5→跳过P2/P3；小目标需要P2/P3→P5可选。层级选择+空间选择=条件计算的双维联合。
- **为什么是Gap**: 
  - YOLO-Master仅做Backbone内的感受野路由（非层级选择），Input-Adaptive DNN做深度早退（全图统一深度）
  - **无人做"per-location的层级选择"**——每个grid cell独立决定需要哪些FPN层级的特征
  - PRNet的迭代深度（阶段0/1/2/3）是层级选择的静态版→频域判据可使其动态化
- **可验证性**: ⏸ 需实验（层级掩码 vs 空间掩码 vs 双维掩码的FLOPs-精度Pareto前沿）
- **关联 Idea**: #5 v3.3(层级+空间双维门控) / #26(层级最优停止·动态扩展)

#### G1-X2 🟠 尺度×OBB：旋转框的尺度-角度联合建模
- **涉及维度**: 🟡尺度 × 🟠OBB
- **描述**: OBB检测中尺度变化和角度变化**不是独立的**——狭长目标（如ship·宽高比5:1）在旋转后尺度感知完全改变（侧视:小目标·俯视:大目标）。当前OBB方法（YOLO26-OBB/FAA/RDCNet）仅单独处理角度或尺度，无人建模"尺度-角度联合分布"。
- **为什么是Gap**: 
  - 联合建模的直接收益: 角度已知时尺度估计更准确（狭长目标的正视图面积=宽×高·侧视图面积=宽×长）→利用角度信息修正尺度估计
  - VALA VIoU在OBB场景的扩展: 旋转虚拟锚框（不仅缩放wh·同时旋转θ）→更精准的旋转框正样本匹配
  - FAA FAE频域角度 + VALA VIoU尺度校准 = 频域驱动的旋转框尺度-角度联合LA
- **可验证性**: ⏸ 需实验（旋转框VIoU vs 标准VIoU在DOTA上的消融）
- **关联 Idea**: VALA→OBB扩展 / FAA×VALA联合 / 新Idea(旋转框双维LA)

#### G1-X3 🟠 尺度×密集遮挡：拥挤场景下的尺度退化与恢复
- **涉及维度**: 🟡尺度 × 🔴密集遮挡
- **描述**: 密集遮挡场景中，目标尺度感知被两个因素破坏: ①遮挡→部分可见→有效尺度减小→被错误分配到更浅层（假阳性小目标）；②邻域目标干扰→边界模糊→尺度估计噪声增大。当前密集遮挡方法（OPL/RepLoss/DALA）和尺度方法（VALA/DERNet）各自独立发展，无人解决"遮挡导致的尺度感知退化"。
- **为什么是Gap**: 
  - 遮挡区域的尺度估计需要特殊处理: 利用可见部分推断完整尺度·或利用上下文（周围目标尺度）做尺度补全
  - DALA的密度判据可修正尺度估计: 高密度区→目标的"有效尺度"可能因遮挡而缩小→用密度信息修正
  - 频域判据可同时感知遮挡和尺度: 高频局部异常度→遮挡边界+高频能量→尺度指示→联合判据
- **可验证性**: ✅ 概念推演·⏸实验验证
- **关联 Idea**: #35(频域遮挡先验→同时服务遮挡和尺度) / #40(密度×尺度双维LA→扩展到遮挡维)

#### G1-X4 🟡 尺度×频域：三维频域判据体系（能量×方向×签名）
- **涉及维度**: 🟡尺度 × 频域判据
- **描述**: 频域判据在项目中的应用正在形成三维体系——①高频能量(#11·内容重要性+尺度指示) ②方向响应(FAA FAE·目标方向) ③频率签名(#25·点/线/面几何形状)。三维共享一次FFT计算→一次计算三次收益。**但三维的联合使用（三维联合决策·而非各自独立）从未被设计。**
- **为什么是Gap**: 
  - 联合决策场景: 高频能量高+方向清晰+点签名→孤立小目标→全算+小kernel；高频能量高+方向混乱+面签名→密集群→全算+混合kernel
  - 三维判据的联合分布（8种组合→8种处理策略）→最高信息利用效率
  - 一次FFT→三维输出的计算共享设计→工程上最高效的频域判据实现
- **可验证性**: ✅ 概念推演（三维判据联合决策表设计），⏸实验验证
- **关联 Idea**: #25(频率签名判据→三维联合体系升级) / #11(高频能量→三维判据一员) / FAA FAE(方向·三维判据一员)

#### G1-X5 🟡 尺度×注意力：尺度感知的注意力范围自适应
- **涉及维度**: 🟡尺度 × 注意力机制
- **描述**: 标准注意力（self-attention/通道注意力/空间注意力）的**感受野范围固定**——与目标尺度无关。小目标需要小范围注意力（避免背景噪声），大目标需要大范围注意力（覆盖完整目标）。尺度感知的注意力范围自适应完全未被探索。
- **为什么是Gap**: 
  - 可变形注意力(DCNv4)的采样点偏移可以尺度感知——高频区→采样点集中→小感受野；低频区→采样点分散→大感受野
  - 频域判据可直接指导注意力范围——高频局部异常度→小kernel注意力·低频均匀→大kernel注意力
  - YOLO-Master MoE已证明"不同尺度目标→不同感受野"有效→扩展到注意力范围自适应是自然下一步
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #5/#11(频域判据扩展·注意力范围选择) / YOLO-Master(MoE参照)

#### G1-X6 🟢 尺度×自监督：尺度不变表征学习的新范式
- **涉及维度**: 🟡尺度 × 表征学习
- **描述**: 对比学习（SimCLR/MoCo/DINO）的尺度不变性来自**数据增强**（随机裁剪+缩放）——隐式而非显式。无人设计**显式尺度不变表征学习**——如多尺度特征的一致性正则化、尺度预测辅助任务、尺度解耦表征。
- **为什么是Gap**: 
  - 检测器的FPN本身是多尺度设计→但FPN各层**无显式尺度一致性约束**——P2的特征和P5的特征对同一目标的表征应满足某种"尺度缩放关系"
  - 尺度解耦表征: 将特征分解为尺度不变部分（语义内容）+尺度可变部分（几何细节）→尺度不变部分用于分类、尺度可变部分用于回归
  - 这是远期理论方向→适合作为学术锚点
- **可验证性**: ✅ 概念推演·⏸实验验证
- **关联 Idea**: #24(信息瓶颈→尺度解耦扩展) / 远期理论Idea

### 一、Scale x Conditional Computation: 频域判据驱动的三维计算分配

**核心交集**: 一次频域判据 -> 三维决策输出

| 维度 | Idea | 决策 | 粒度 |
|------|------|------|------|
| 空间门控 | #5/#11 | 算/不算(二值) | per-position |
| 层级门控 | #5 v3.3 (G1-X1) | 需要哪些FPN层(多选) | per-position x per-layer |
| 专家路由 | #43 (G1-S1) | 用哪个专家(多选) | per-position |

**五类区域联合策略**:

| 频域特征 | 空间 | 层级 | 路由 | 联合决策 |
|---------|------|------|------|---------|
| 高频+点状(小目标) | 算 | P2+P3全算,P4+P5可选 | 小kernel(3x3) | 全算+浅层+小感受野 |
| 高频+线状(边缘) | 算 | P2+P3为主,P4辅助 | 中kernel(5x5) | 算+中层+中感受野 |
| 中频+面状(中大) | 算 | P4+P5全算,P2+P3跳 | 大kernel(7x7) | 算+深层+大感受野 |
| 低频+均匀(背景) | 跳 | 全部跳过或P5轻量 | 无 | 零计算 |
| 高频+混乱(密集) | 保留 | P2+P3全算(小目标保护) | 混合专家 | 全算+浅层+混合 |

**协同增益**: (1)决策一致性--空间跳 -> 层级和路由自动失效 (2)频域判据摊薄 1:1 -> 1:3 (3)消融可拆分(渐进式论文: 空间 -> +层级 -> +路由) (4)PRNet静态迭代 -> 频域动态化

**张力**: (1)粒度不匹配--P2空间 x FPN多层 x Backbone各stage (2)级联梯度冲突--YOLO-Master已发现路由冲突 (3)负载均衡恶化--频域偏斜导致某些组合占比过高

### 三、Scale x Dense Occlusion: 频域判据统一的尺度退化恢复

**核心交集**: 遮挡破坏尺度感知 <-> 频域同时感知遮挡+尺度

遮挡的双重破坏: (a)部分可见 -> 有效尺度减小 -> 错配浅层 (b)邻域干扰 -> 边界模糊 -> 尺度估计噪声增大
频域的双重修复: (a)高频局部异常度 -> 遮挡边界(#35) -> "这里被遮挡了,实际尺度更大" (b)频域局部统计 -> 真实拥挤度(#40) -> "密度高,O2O匹配"

**遮挡-尺度退化-恢复完整管线**:
S1判据(一次计算) -> 遮挡图o(x,y)+尺度图s(x,y)+密度图d(x,y)
-> 联合修正: 尺度修正(s_corrected = s x (1+alpha*o) / LA修正(K(rho) -> K(rho,o)) / NMS修正(sigma -> sigma(density,o))
-> 三线下游: 修正尺度->层级门控(#5 v3.3) / 修正LA->标签分配(#40 v2.0) / 修正NMS->后处理(#38)

**D1 v2.0升级路径**:

| 组件 | v1.0(当前) | v2.0(Scale x Occlusion后) | 升级点 |
|------|-----------|--------------------------|--------|
| 上游判据 | S1空域高通代理 | S1+尺度感知归一化(按频段加权) | 尺度自适应 |
| 下游1 #35 | 频域遮挡先验(固定阈值) | 尺度感知遮挡检测(小目标区降阈值) | 尺度-遮挡联合 |
| 下游2 #38 | 频谱感知NMS(统一sigma_c) | 尺度+遮挡联合衰减(小+遮挡->最大sigma) | 三维联合NMS |
| 下游3 #40 | 连续密度LA | 遮挡修正密度(部分可见->密度向下修正) | 遮挡-密度解耦 |

**张力**: (1)#5跳过vs遮挡保留: 遮挡区高熵 != 不重要 -> 三态门控(#36: 低熵全算/高熵x遮挡->保护算/高熵x非遮挡->跳过) (2)尺度修正方向性: 需"可逆性"约束 (3)频域判据三重负担 -> 需多尺度频域分析

**新高分交叉Idea**:
- D1 v2.0: 尺度-遮挡-密度三维统一框架(一次频域判据->三维输出->联合修正LA+NMS+门控) ~4.3 | #35+#38+#40+#5

## 🟠 OBB 旋转检测 G1 Gap 分析（2026-07-19）

> 基于 OBB P0 深读 3 篇（FAA CVPR 2026 / YOLO26-OBB arXiv 2026.06 / RDCNet IEEE JSTARS 2026.04）+ OBB L1 检索 21 篇 + OBB K1 知识补充（loss/head/augmentation 三文件 OBB 专节），按三模块系统分析：跨P0交叉空白 / L1未覆盖维度 / OBB×条件计算×密集×频域×尺度交叉Gap。每个Gap标注严重程度（🔴致命/🟠重要/🟡机会/🟢远期）、可验证性（⏸需实验/✅可文档推演）、关联Idea、优先级。

### 三角定位：三篇P0的差异化覆盖

| 维度 | FAA (CVPR 2026) | YOLO26-OBB (2026.06) | RDCNet (JSTARS 2026.04) |
|------|-----------------|---------------------|------------------------|
| **角度来源** | 频域FFT→物理先验 | 空间域直接回归→工程最优 | 极坐标DCN→特征学习 |

| **NMS策略** | 标准旋转NMS | NMS-free (O2O) | 标准旋转NMS |

### 一、跨P0交叉空白（FAA × YOLO26-OBB × RDCNet）

#### G1-O1 🔴 频域角度 × 空间域角度的双源监督融合
- **描述**: FAA从频域读角度（物理先验·免学习·单峰假设）、YOLO26-OBB从空间域回归角度（数据驱动·可学习·最优工程精度）——两套角度估计范式完全独立，从未被联合。**一次FFT同时输出能量(尺度指示·#11)和方向(角度指示·FAA FAE)→同一次频域变换服务两条下游。**
- **为什么没人做**: FAA 2026年6月(CVPR)刚发表；频域角度估计在YOLO社区几乎无人涉足；两阶段vs单阶段的架构鸿沟
- **具体方案**: FAE角度作为YOLO26-OBB角度回归的辅助监督（`L_angle = L_reg + α·L_freq_consistency`）或初始化先验（FAE θ̂→角度分支bias初始化）→零推理开销
- **可验证性**: ⏸ 需实验（双源角度损失在DOTA上的消融）
- **关联 Idea**: #44(OBB双维LA·FAA先验) / #17(角度表示选型·FAA提供对照臂) / 新Idea(双源角度监督)
- **优先级**: P0 · 严重度: 🔴致命(角度=OBB核心·双源互补是差异化创新的最佳入口)

#### G1-O2 🟠 NMS-free (YOLO26) vs 频域NMS加速 (FAA) 的路线对比空白
- **描述**: YOLO26-OBB选择NMS-free(O2O一对一匹配·消除NMS)；FAA的FAE角度可作为NMS加速器(两框θ̂差异大→跳过旋转IoU)。两条路线从未被系统对比：O2O在密集OBB场景的recall瓶颈(K=1) vs 频域NMS在极端密集场景的角度估计噪声。**第三条路: NMS-free + 频域后验证**(O2O先保证端到端→频域角度相似度作为第二意见过滤误匹配)。
- **为什么没人做**: NMS-free和频域NMS加速分属不同社区(YOLO vs MMRotate)；YOLO26-OBB 2026年6月刚发布
- **具体方案**: 三臂对照实验——①纯NMS-free(O2O K=1) ②NMS-free+频域角度后验证(K=1→θ̂过滤) ③频域加速NMS(标准NMS+FAE预筛)→DOTA密集子集上对比
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #38(频谱感知Soft-NMS·OBB扩展) / #41(密度自适应NMS·O2K) / 新Idea(频域角度后验证)
- **优先级**: P0 · 严重度: 🟠重要(NMS=OBB推理瓶颈·路线选择影响后续所有OBB工作)

#### G1-O3 🟠 旋转等变三范式在YOLO的迁移可行性系统对比空白
- **描述**: 三种旋转等变范式已在非YOLO检测器验证——①群卷积(ReDet·最昂贵·DOTA 76.25) ②Hilbert曲线(HERO-Det AAAI 2026·最复杂·DOTA 79.56) ③极坐标DCN(RDCNet·最简洁·DOTA 81.37)——但**三者在YOLO架构上的迁移难度/精度增益/速度损失从未被系统比较**。
- **为什么没人做**: 三者分别来自不同团队/不同检测器家族；YOLO社区对旋转等变的关注度低（更关注角度表示和Loss改进）
- **具体方案**: 以YOLO26-OBB为统一基线，分别嵌入RDC(极坐标DCN→C2f)、HERO(Hilbert遍历→SPPF替换)、群卷积(C4→C2f conv替换)→同基准三臂消融→产出"YOLO旋转等变选型指南"（可独立成文）
- **可验证性**: ⏸ 需实验（但可先文档推演三范式的YOLO伪代码集成点）
- **关联 Idea**: RDCNet→YOLO迁移 / HERO-Det→YOLO迁移 / 旋转等变选型指南(新Idea)
- **优先级**: P1 · 严重度: 🟠重要(旋转等变=OBB长期方向·选型指南有独立发表价值)

#### G1-O4 🟡 极坐标(ρ,θ)的频域-空间域统一数学框架
- **描述**: FAA FAE和RDCNet RDC都使用了(ρ,θ)极坐标参数化，但语义完全不同——FAA从频谱读方向(物理先验·免学习)、RDCNet从特征学方向(可学习DCN偏移)。**两者共享(ρ,θ)参数空间→可建立统一的"旋转感知极坐标框架"：FAA判据初始化`Δρ₀, Δθ₀`→RDC可学习偏移`Δρ, Δθ`在此基础上微调→物理约束引导学习。**
- **为什么没人做**: FAA和RDCNet分别从频域和空间域出发，跨域统一需要数学建模（非简单拼接）
- **具体方案**: 理论分析→"频域和空间域在极坐标下的同构性"→FAA先验→RDC初始化→收敛加速+泛化提升的假设+验证
- **可验证性**: ⏸ 需实验（但数学推导部分可文档推演）
- **关联 Idea**: RDCNet→FAA联合 / 新Idea(物理约束引导旋转感知学习)
- **优先级**: P2 · 严重度: 🟡机会(长期理论价值·短期发表窗口较远)

#### G1-O5 🟠 全链路旋转感知联合设计空白
- **描述**: RDCNet在Backbone编码旋转信息→经标准PAFPN(无旋转感知)→旋转信息稀释。FAA在Neck做方向对齐→但Backbone无旋转感知特征。YOLO26-OBB仅在Head处理角度。**三层割裂: Backbone旋转感知(RDC) + Neck方向融合(FAAFusion) + Head旋转解耦(FAA Head/NMS-free)从未被联合设计。**
- **为什么没人做**: 三个模块分别在三篇独立论文中，无人做跨论文的联合设计
- **具体方案**: RDC轻量版(C2f嵌入) + FAAFusion轻量版(PAN插入·方向感知加法) + YOLO26-OBB NMS-free Head → 全链路旋转感知YOLO-OBB
- **可验证性**: ⏸ 需实验
- **关联 Idea**: FAA×RDCNet联合 / 新Idea(全链路旋转感知YOLO)
- **优先级**: P0 · 严重度: 🟠重要(全链路联合=最大的精度天花板·但工程复杂度高)

#### G1-O6 🟡 AALA vs SimOTA/TAL 在OBB场景的系统对比
- **描述**: RDCNet的AALA(宽高比无关centerness+几何感知采样+任务对齐评分) vs YOLO26-OBB的STAL+SimOTA vs 标准TAL——三种OBB标签分配策略从未被系统对比。AALA的"宽高比无关"主张对狭长目标(船/桥)理论上最优，但在YOLO SimOTA框架下是否仍然成立？
- **为什么没人做**: AALA在MMRotate框架验证；SimOTA/TAL在YOLO框架验证；跨框架对比的技术壁垒
- **具体方案**: 将AALA的宽高比无关centerness移植到YOLO26-OBB SimOTA中→对比原始SimOTA→验证AALA增益是否跨框架
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #40(AALA→双维LA密度+宽高比) / #12(KLD分配→OBB场景重新验证)
- **优先级**: P2 · 严重度: 🟡机会(AALA理论优雅·但迁移难度中等)

### 二、L1未覆盖维度（OBB L1 21篇·P1×4 + P2×14）

#### G1-L1 🟠 BD Loss→OBB标签分配的直接迁移
- **描述**: BD Loss(Bhattacharyya距离)全面超越GWD/KLD/KFIoU——满足IoU全部理想属性(非负/对称/三角不等式/尺度不变)。但BD Loss仅作为回归Loss使用，从未被用作标签分配度量（如#12 KLD分配）。**BD Assign = BD距离替代IoU作为SimOTA匹配代价**。
- **为什么没人做**: BD Loss 2025年刚提出；社区还停留在"替换回归Loss"阶段，未想到分配度量
- **具体方案**: SimOTA匹配代价 `C = BD(b_pred, b_gt) + λ·C_cls`→对比KLD分配和标准IoU分配
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #12(KLD分配→BD升级) / 新Idea(BDAssign)
- **优先级**: P1 · 严重度: 🟠重要(分配度量=低悬垂果实·可直接复用BD Loss代码)

#### G1-L2 🟡 HERO-Det Hilbert旋转等变→YOLO C2f嵌入
- **描述**: HERO-Det(AAAI 2026)的Hilbert曲线遍历卷积是群卷积的轻量替代，但当前实现复杂（Hilbert曲线映射+pixel shuffle）。YOLO C2f模块的标准3×3卷积→替换为Hilbert遍历卷积→保留空间局部性+旋转等变性。
- **为什么没人做**: HERO-Det刚发表(AAAI 2026)；Hilbert曲线在YOLO中的嵌入方式需要适配C2f的split-concat结构
- **具体方案**: C2f-Hilbert = Bottleneck中3×3 DWConv替换为Hilbert-curve DWConv（仅改变遍历顺序·零额外参数）→与RDCNet极坐标DCN形成对照
- **可验证性**: ⏸ 需实验
- **关联 Idea**: G1-O3(三范式对比) / 旋转等变选型指南
- **优先级**: P2 · 严重度: 🟡机会(Hilbert新颖但工程复杂)

#### G1-L3 🟡 SFMP-Net空域-频域融合→OBB频域判据双向增强
- **描述**: SFMP-Net(2026)在OBB上验证了空域-频域动态增强的有效性——但其频域用法是传统的"增强"而非"判据"。**将其频域模块改造为判据输出**: 原来的"频域特征×空域特征→增强特征"改为"频域特征→空间重要性图→条件计算门控"。同一模块两种用途的对比=增强vs判据在OBB场景的路线裁决。
- **为什么没人做**: 频域判据是项目独创方向，OBB社区全是增强范式（与DETR/HBB频域浪潮一致）
- **具体方案**: SFMP-Net频域分支→拆分为两条路径——①增强路径(原论文) ②判据路径(#11式空间门控)→DOTA消融对比→"OBB频域增强 vs OBB频域判据"路线裁决
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #11(OBB扩展) / #45(频域门控路线裁决·OBB场景实例化)
- **优先级**: P2 · 严重度: 🟡机会(OBB频域判据空白确认·与HBB/DETR一致)

#### G1-L4 🟢 YOLO-OBB轻量化路线图空白
- **描述**: GADet(20.3GFLOPs/56.5FPS·Pareto前沿)、EdgeYOLO-RS(边缘部署)、YOLO11n-OBB改进(100FPS/2.71M)——已有多个YOLO-OBB轻量化工作，但**缺乏统一的轻量化路线图**：①Backbone轻量(EdgeYOLO) ②Head轻量(YOLO26 NMS-free) ③后处理轻量(频域NMS加速·G1-O2) 三者联合的Pareto前沿分析空白。
- **为什么没人做**: 轻量化工作各自独立发表，缺乏系统性综述/对比框架
- **具体方案**: 三维Pareto分析(Backbone×Head×后处理)→"YOLO-OBB轻量化路线图"综述
- **可验证性**: ✅ 可纯文档推演（基于已发表数字做meta-analysis）
- **关联 Idea**: OBB基线选型(影响#6/#17的OBB基线选择)
- **优先级**: P2 · 严重度: 🟢远期(综述性质·独立发表价值有限)

#### G1-L5 🟢 OBB数据增强的旋转等变约束理论
- **描述**: NBBOX(标注级噪声注入)和PointOBB-v3(对称性SSL)分别从监督和自监督角度做OBB数据增强——但**无人分析数据增强操作本身的旋转等变性**。旋转→增强→检测器的信息流中，增强步骤的旋转等变性是否被保持？破坏了会导致什么？
- **为什么没人做**: 数据增强通常被视为工程trick而非理论问题；旋转等变约束→数据增强的理论分析是交叉空白
- **具体方案**: 理论框架——"OBB数据增强的旋转等变损失"→定义增强前后旋转信息的保持度→分析现有增强策略的保持度→提出等变保持增强策略
- **可验证性**: ✅ 可部分文档推演（理论分析+现有数字meta-analysis）
- **关联 Idea**: augmentation.md OBB专节扩展
- **优先级**: P2 · 严重度: 🟢远期(理论贡献·但发表周期长)

#### G1-L6 🟠 YOLO-OBB密集场景的NMS-free vs Soft-NMS路线裁决

- **为什么没人做**: OBB密集场景数据集少（DOTA密集子集≠VisDrone密集）；NMS-free刚成为YOLO OBB主流
- **具体方案**: ①O2K扩展——基于#40连续密度LA估计每个grid cell的目标密度→动态K ②OBB Soft-NMS——基于FAA FAE角度相似度(非IoU)做衰减→"角度感知Soft-NMS"
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #40(O2K扩展) / #38(OBB Soft-NMS) / #41(密度自适应O2K)
- **优先级**: P0 · 严重度: 🟠重要(OBB×密集=项目两个战略维度的交汇点)

#### G1-L7 🟡 频域OBB三子方向的判据化改造

- **为什么没人做**: 条件计算+OBB=全新交叉维度，此前无人触及
- **具体方案**: ①#5 P2门控在OBB场景的grid cell覆盖率分析（HBB vs OBB grid cell分布差异）②#11高频能量判据在旋转框上的响应模式（旋转→频谱旋转？）③理论分析→是否需要OBB专属判据

##### G1-X2 🟠 方向模糊区的条件计算策略
- **描述**: 宽高比≈1的目标（正方形·如车辆/储罐）→方向模糊→角度不确定性高→**既是检测难点也是计算节省机会**：方向不确定的区域→高频能量均匀分布(无方向峰值)→与方向明确的狭长目标(桥/船·方向峰值明显)的频域响应完全不同→方向模糊度可作为条件计算判据的第四维。
- **为什么没人做**: 方向模糊区=OBB独有问题·HBB无此维度·条件计算社区未涉足OBB
- **具体方案**: FAA FAE的角度熵(θ分布的离散度)→高方向熵区(正方形目标)→激进稀疏化(反正角度也难回归)→低方向熵区(狭长目标)→保留计算(需要精准角度)
- **可验证性**: ⏸ 需实验（FAA FAE的角度熵在DOTA上的分布分析）
- **关联 Idea**: #11(OBB·方向维度) / #25(频率签名·正方形vs线状签名差异) / 新Idea(方向模糊度门控)
- **优先级**: P1 · 严重度: 🟠重要(OBB条件计算的独特切入点·差异化强)

#### OBB × 密集遮挡

##### G1-X3 🔴 旋转框密集场景的NMS问题比HBB更严重
- **描述**: ①旋转IoU计算=O(nlogn)凸多边形求交→单次NMS比HBB贵10-100× ②OBB密集场景中旋转框IoU重叠理论上>HBB(框更"紧实"→相邻目标更容易重叠) ③D1框架的三线共享(#35遮挡先验/#38频谱NMS/#40密度LA)全部基于HBB设计→OBB扩展是D1框架的自然延伸和压力测试。
- **为什么没人做**: OBB密集场景本身研究极少（DOTA密集子集未标准化）；D1框架刚完成·OBB扩展是天然下一步
- **具体方案**: ①D1→OBB-D1：频域判据驱动OBB密集检测统一框架（S1判据计算→θ̂+FAA角度→三条OBB下游）②OBB遮挡图生成：频域遮挡先验+旋转框自适应 ③OBB频谱NMS：角度相似度(FAA FAE) × 频谱相似度(#38)联合衰减
- **可验证性**: ⏸ 需实验
- **关联 Idea**: D1 v2.0(OBB扩展) / #38(OBB NMS) / #35(OBB遮挡图) / #40(OBB O2K)
- **优先级**: P0 · 严重度: 🔴致命(OBB×密集=两个战略维度的最强交汇·D1框架OBB扩展=最高项目协同)

##### G1-X4 🟠 旋转框下的遮挡图生成——OPL范式OBB迁移
- **描述**: OPL的bbox重叠→遮挡图生成在HBB中有效——但OBB中旋转框重叠的几何定义更复杂：两个旋转框的"遮挡程度"不仅是重叠面积，还与**遮挡方向**有关（同向船并排 vs 异向船交叉→同样的IoU但遮挡严重度不同）。频域判据可绕过旋转框重叠计算→直接从特征图读遮挡程度（#35频域遮挡先验·OBB版）。
- **为什么没人做**: OPL是HBB遮挡感知+ESWA 2025；OBB遮挡感知无人涉足
- **具体方案**: #35 OBB扩展——P2特征图→空域高通代理→高频异常区→旋转框内高频异常度=遮挡指标→替代旋转IoU重叠计算→OBB OPL（Oriented OPL）
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #35(OBB扩展) / 新Idea(Oriented OPL)
- **优先级**: P1 · 严重度: 🟠重要(OPL→OBB=低悬垂果实)

#### OBB × 频域

##### G1-X5 🔴 一次FFT→尺度+方向双输出：OBB三维频域判据族
- **描述**: 同一次FFT变换输出三个维度信息——①高频能量(标量·目标重要性·#11) ②方向响应(矢量·FAA FAE角度·#44) ③频率签名(矩阵·几何形状·#25)——三者在HBB场景已独立验证价值。**在OBB场景中三者天然联合——同一目标的一次FFT同时给出"是否重要(能量)"+"哪个方向(角度)"+"什么形状(签名)"→OBB三维频域判据族。**
- **为什么没人做**: 三者分属三个独立研究方向（能量=条件计算/角度=FAA/签名=#25理论），从未被同一框架统一
- **具体方案**: OBB三维频域判据框架——①能量→OBB条件计算门控(#11 OBB扩展) ②方向→OBB角度先验(R-VIoU #44) + OBB NMS角度相似度 ③签名→宽高比先验(线状=狭长目标·点状=正方形) + 方向模糊度(G1-X2)→一次FFT三次收益·OBB场景收益>HBB
- **可验证性**: ⏸ 需实验（但三维度独立可消融·最低成本先验AUROC验证→实验模块最低门槛）
- **关联 Idea**: #11(OBB扩展) / #25(OBB扩展) / #44(FAA角度) / 新Idea(OBB三维频域判据框架)
- **优先级**: P0 · 严重度: 🔴致命(频域=项目核心技术资产·OBB三维判据族=频域价值最大化的最佳场景)

##### G1-X6 🟠 频域判据→OBB NMS角度相似度加速
- **描述**: OBB NMS的核心瓶颈是旋转IoU计算昂贵（凸多边形求交·O(nlogn)）。FAA FAE提供一种绕过方法: ①两框FFT角度估计θ̂_A, θ̂_B → ②|θ̂_A − θ̂_B| > τ_angle → 方向差异大→直接跳过旋转IoU(大概率不重叠) → ③仅对方向接近的框对计算旋转IoU。**方向过滤是旋转IoU的廉价代理，理论加速比取决于τ_angle与场景的联合分布。**
- **为什么没人做**: FAA FAE从未被提出用于NMS加速（原用途是检测头方向对齐）
- **具体方案**: ①理论分析——DOTA/VisDrone上方向差异|Δθ̂| vs 真实IoU的联合分布→设定τ_angle的保守阈值 ②实验——FAA加速NMS vs 标准NMS vs NMS-free的速度-精度trade-off
- **可验证性**: ⏸ 需实验（但方向差异→IoU联合分布分析可纯数据推演·无需训练）
- **关联 Idea**: #38(OBB NMS加速) / FAA FAE→NMS加速(新应用)
- **优先级**: P1 · 严重度: 🟠重要(低悬垂果实·FAA FAE已开源·理论分析+统计验证可纯CPU完成)

#### OBB × 尺度

##### G1-X7 🟠 旋转框尺度-角度联合建模（深化Scale G1-X2）
- **描述**: Scale G1-X2已识别"尺度-角度联合"为交叉Gap。此处深化：①狭长目标（宽高比>3:1）→角度变化→有效尺度变化（侧视=s·宽高比×s→小目标/俯视=s→大目标）→同一目标在不同角度下的尺度可变化5-10× ②尺度感知的虚拟锚框(VALA VIoU)在OBB中需同时校准wh和θ→"旋转虚拟锚框"=VIoU的OBB自然扩展 ③YOLO26-OBB的STAL在旋转框膨胀时角度保持未讨论→膨胀+旋转的联合优化。
- **为什么没人做**: Scale G1-X2刚识别；尺度-角度联合建模需要三维参数空间(s, w/h, θ)的联合统计→计算复杂度高
- **具体方案**: ①R-VIoU = VIoU(wh校准) + 方向校准(θ̂→每角度bin虚拟锚框)→旋转框双维LA ②FAA FAE角度先验→方向维虚拟锚框→免GT统计（与#44协同）
- **可验证性**: ⏸ 需实验
- **关联 Idea**: #44(OBB双维LA) / VALA→OBB扩展 / FAA+VALA联合
- **优先级**: P1 · 严重度: 🟠重要(LA在OBB的独特机会·但依赖Scale G1-X2的框架)
