## 1. 问题 (Problem)
遥感图像的层级标签结构中，目标分布在多个粒度层级（如：舰船 → 军舰 → 尼米兹级航母）。传统层级分类要求**强制叶节点预测（MLNP）**——模型必须预测最细粒度类别。这导致：
1. **无法处理未见类别**：推理过程不能停在较粗粒度上
2. **长尾/不确定目标被强制误分**：低置信度的细粒度预测无法优雅地降级为粗粒度预测

## 2. 方法 (Method)

### 2.1 多粒度分类（MGC）框架
首次将 MGC 引入遥感图像——模型可预测标签层级中**任意层级**的类别，而非仅叶节点。

### 2.2 Accuracy-Specificity 曲线
- 沿标签树自顶向下预测：降低阈值 → 增加 specificity（更细粒度）但降低 accuracy
- 曲线定量衡量两个维度的 trade-off，**自适应选择**任务相关的阈值
- 类比目标检测中的 Precision-Recall 曲线——用曲线下面积（AC/AP）作公平比较

### 2.3 语义距离约束 Loss
- 在 label hierarchy 中定义两节点的**语义距离**：沿树路径的边数
- 将其作为 logit adjustment 的惩罚项融入 CE loss：离 GT 节点越远的类别，logit 被压制越大
- 关键改进：标准距离 `distance(y, y')` 无法区分"兄弟节点"和"非兄弟节点"的预测质量差异 → 使用**考虑 LCA 的修正语义距离** `sem_dist(y, y')`

### 2.4 有序 Pareto 集推理
- 推理时按概率↓ + 信息量↑构建有序 Pareto 集
- 根据用户指定的 accuracy/specificity 要求选择阈值 → 在 Pareto 集中选择满足阈值且信息量最高的节点作为预测

### AC / AP（Accuracy-Specificity 曲线下面积）
- 在 HRSC / FGSC / FGSRSIS 三个数据集上均达到**最外围边界**（即最佳 AC 和 AP）
- 在 FGSRSIS（最复杂的层级结构）上优势尤为显著

### Zero-shot 分类
- 随机移除部分细粒度子类 → 模拟"未见类别"场景
- 在大多数指标上排名第一，在 FGSRSIS 上唯一达到最外围边界

## 4. 创新点分析 (≥3)
1. **首次将 MGC 引入遥感**：填补遥感图像中"多粒度分类"的空白——此前工作均局限于叶节点分类
2. **语义距离约束 Loss**：用标签树中任意两节点的全局语义距离（非仅父子/兄弟关系）作 logit 正则化的惩罚项
3. **Accuracy-Specificity 曲线**：借鉴 PR 曲线的哲学——用连续曲线替代固定阈值评估 MGC，适应不同任务需求
4. **有序 Pareto 集推理**：多目标优化（概率高 + 信息量大）的优雅形式化

## 5. 弱点 (≥5)
1. **纯分类任务**：不涉及目标检测（无 bounding box 预测）——与检测 pipeline 的交叉有限
2. **依赖预定义层级结构**：需要人工构建的 label tree——标签层级质量影响性能
3. **仅遥感舰船验证**：HRSC/FGSC/FGSRSIS 均为舰船细粒度分类——其他遥感目标（飞机、车辆）未验证
4. **代码未公开**
5. **语义距离定义简单**：仅基于树中边数 → 未考虑不同层级间的语义差异可能非均匀
6. **计算开销**：推理时需构建 Pareto 集 + 曲线遍历 → 比标准分类慢
7. **TGRS 期刊**（IF 5.6）：非 CVPR/ICCV/NeurIPS 顶会级别

## 6. 对本项目的启发

### 概念层面的跨界映射
"自适应粒度选择"的核心思想——在不同场景下选择不同粒度的输出——在检测领域有对应概念：

| MGC (本文) | 对应的检测概念 |
|------------|--------------|
| 叶节点预测 = 最细粒度 | 大目标检测（高分辨率、精确定位） |
| 中间节点预测 = 较粗粒度 | 小目标检测（低分辨率、粗略定位即可） |
| 自适应选择停止层级 | 自适应选择检测层（P2 vs P3 vs P4）？ |

### 对标签分配的间接启发
- 本文的"语义距离 penalty"→ #12 的标签分配中，不同尺度的 GT box 可能对应不同的"分配粒度"
- "远离 GT 的节点被压制" → "与 GT 尺度不匹配的 anchor 被压制"的软版本

### ⚠️ 直接适用性评估
- 本文是**分类**任务，不涉及 bounding box 回归
- "自适应粒度选择"概念在检测中的物理含义完全不同（层级粒度 vs 空间粒度）
- 不建议强行建立关联——本文的核心价值在于**方法论意识**（"粒度应该自适应"）而非具体技术方案
