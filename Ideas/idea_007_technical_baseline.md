### 1.2 MOCHA 的可复用组件

| 组件 | 复用度 | 说明 |
|------|--------|------|
| Translation Module 架构 | ✅ 直接复用 | 通道级 MHA + MLP, 轻量有效 |
| L_dist + L_emb 损失框架 | ✅ 直接复用 | ℓ₁+ℓ₂ 距离 + 关系保持 |

## 二、Idea#7 差异化：熵加权蒸馏

### 2.1 核心改动：一行公式的区别

```
MOCHA (均匀权重):
  L_dist = (1/N) * Σ_i |f_s'(i) - f_t(i)|

Idea#7 (熵加权):
  L_dist = Σ_i w_i * |f_s'(i) - f_t(i)| / Σ_i w_i
  其中 w_i = H_i / H_mean  (归一化语义熵)

语义熵 H_i 的计算：
  - 对 GT 框 i 内的区域提取 YOLOE/CLIP 嵌入
  - 计算与类别文本的相似度分布熵
  - H 高 → 模型对该目标"不确定" → 需要更多蒸馏信号
  - H 低 → 模型对该目标"确定" → 可降低蒸馏强度
```

### 2.2 为什么熵加权比均匀权重更好？

| 维度 | 均匀蒸馏（MOCHA） | 熵加权蒸馏（#7） |
|------|-------------------|------------------|
| 蒸馏重点 | 所有 GT 等权 | **困难样本高权重** |
| 小目标 | 同等对待（但特征弱→蒸馏效果差） | **天然高熵 → 高蒸馏权重** |
| 大目标 | 同等对待（但特征强→蒸馏饱和） | 天然低熵 → 低蒸馏权重（避免过拟合） |
| 训练效率 | 均匀→收敛慢 | **聚焦困难区域→更快收敛** |
| 语义偏差 | 均匀→偏差被平均隐藏 | **熵加权→偏差显式处理（温度缩放校正）** |

### 2.3 关键假设（需要实验验证）

> **H1**: 小目标区域的语义熵系统性地高于大目标区域
> → 如果成立：熵加权蒸馏天然利好小目标（自动提升小目标学习权重）

> **H2**: 高熵区域的 MOCHA 蒸馏梯度更大（student-teacher 差异更大）
> → 如果成立：熵加权→聚焦"最需要蒸馏的区域"→效率更高

> **H3**: 熵加权比均匀权重在 VisDrone 小目标上 AP_s 提升 ≥ 1.0
> → 目标：证明"熵信号在蒸馏任务中具有信息价值"

## 三、实验设计

### 3.1 基线设置

| 组件 | 选择 | 理由 |
|------|------|------|
| Student | YOLO11n | 项目标准基线 |
| Teacher | YOLOE-S (CLIP嵌入) | 比 LLaVa-7B 轻 100×, 可单卡训练 |
| Teacher特征 | YOLOE 物体嵌入 (text-aligned) | 天然适合语义熵计算 |
| 数据集 | VisDrone | 项目主 benchmark |
| 蒸馏区域 | GT 框 + 高质量 proposal | MOCHA 原用 GT, 可扩展到 proposal |
| Translation Module | 通道 MHA + 2层 MLP | MOCHA 原设计 |

### 3.2 消融实验

| 实验 | 描述 | 验证目标 |
|------|------|----------|
| **E0** | YOLO11n + SLE baseline (#6) | 无蒸馏基线 |
| **E1** | E0 + MOCHA 均匀蒸馏 | 复现 MOCHA→确认蒸馏基线收益 |
| **E2** | E1 + 语义熵加权 (H-based) | **核心实验** → 熵加权 > 均匀？ |
| **E3** | E2 + 温度缩放校正 (τ_temp) | CLIP-Bias 校正是否提升？ |
| **E4** | E2 + 仅蒸馏小目标 (GT area < 32²) | 消融：收益来自熵加权 or 仅关注小目标？ |
| **E5** | E2 + 高频局部异常度加权 (对比) | #11 判据 vs #5 判据在蒸馏中的效果 |

### 3.3 对照实验

| 实验 | 描述 | 目标 |
|------|------|------|
| **C1** | 随机加权 (shuffle H) | 证明熵不是"任意加权都行" |
| **C2** | 反熵加权 (w = 1/H) | 低熵区域高权重→预期掉点（证明方向性） |
| **C3** | GT 面积加权 (w ∝ 1/area) | 对比熵加权 vs 简单的尺度加权 |
| **C4** | Oracle 加权 (GT AP 改善空间) | 理论上限：最需要蒸馏的区域 |

## 四、技术路线图

```
Phase 1: 再现 MOCHA 基线（1-2天）
  - 搭建 Translation Module + 蒸馏 loss
  - YOLO11n + VisDrone 复现 MOCHA 蒸馏框架
  - 验证：蒸馏 > 无蒸馏 (Δ > 0)

Phase 2: 熵加权蒸馏（2-3天）
  - 实现语义熵计算模块（基于 YOLOE 物体嵌入）
  - 实现熵加权 loss
  - 验证：熵加权 > 均匀加权 (Δ > 0.5 AP_s)

Phase 3: CLIP-Bias 校正（1天）
  - 添加温度缩放校正
  - 验证：校正 > 未校正 (Δ > 0.2 AP_s)

Phase 4: 分析与投稿（2-3天）
  - 消融 + 对照实验完成
  - 可视化：熵图 vs 蒸馏梯度分布
  - 论文写作

总计：6-9天（需 GPU）
```

## 五、风险与缓解

| 风险 | 级别 | 缓解 |
|------|------|------|
| MOCHA 在 VisDrone 上不收敛 | 🟡 3/5 | MOCHA 原在 few-shot 场景验证→通用检测可能需调参 |
| 熵加权效果不显著 (Δ < 0.5) | 🟡 3/5 | 预实验先验证 H1/H2 假设→不成立则放弃 #7 |
| Teacher 在 VisDrone 上精度不足 | 🟡 3/5 | YOLOE-S VisDrone 零样本精度未知→可能需微调 |
| CLIP 偏差→熵加权方向错误 | 🟡 3/5 | E3 温度缩放 + C2 反熵加权（确认方向） |
| Translation Module 训练不稳定 | 🟢 2/5 | MOCHA 已验证稳定→直接复用超参 |

### 终止条件

如果 Phase 1 无法复现（蒸馏无收益）或 Phase 2 熵加权 vs 均匀加权无显著差异 → **放弃 #7，资源集中到 #5/#11**。
