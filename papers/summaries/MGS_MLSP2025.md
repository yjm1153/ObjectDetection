# MGS: Micro-Gated Sparsification — Utilizing Dynamic Sparsity on Pretrained DETR

> MLSP 2025 (IEEE International Workshop on Machine Learning for Signal Processing) | Reza Sedghi, Anand Subramoney, David Kappel (Bielefeld University / Royal Holloway, University of London)
> 代码: 未公开; arXiv: 2510.09380

---

## 1. 问题 (Problem)
DETR 的 MLP 层（FFN）中 ReLU 激活后存在 **高达 97% 的平均激活稀疏性**，但这些"死神经元"是**输入依赖的**（动态变化），无法通过静态剪枝利用。已有的静态启发式方法（SIBS）仅能实现 ~20% 稀疏率，实用性有限。

## 2. 方法 (Method)

### 2.1 Micro-Gate 结构
在每个 MLP 块的第一个线性层**之前**插入一个紧凑门控模块：
- **结构**：单层线性层 + Sigmoid 激活
- **规模**：门控神经元数 ≈ 原始 MLP 层神经元数的 **12%**
- **分组控制**：每个门控神经元控制原始层中的**一组神经元**（而非一一对应）
- **输入**：与原始线性层相同的输入特征

### 2.2 训练方式
- **Backbone 完全冻结**：仅训练门控参数
- **监督信号**：对每组神经元计算其激活的 **Euclidean Norm**；norm > 0 → label=1（活跃），否则 label=0（非活跃）
- **损失函数**：Binary Cross-Entropy（多标签分类）
- **训练量**：COCO 2017 上仅 **5 epochs**

### 2.3 推理时稀疏化
- Sigmoid 输出经阈值（默认 0.5）二值化 → 低于阈值的神经元组被**整组跳过**
- 跳过传播到两个线性层：第一层跳过对应权重列，第二层跳过对应权重行
- 实现**真实 FLOPs 减少**（非仅内存节省）

## 3. 关键实验结果

| 配置 | 稀疏率 | mAP |
|------|--------|-----|
| Vanilla DETR | 0% | 0.399 |
| MGS (单层) | 85–95% | 维持或略超 baseline |
| MGS (全层, uniform threshold=0.5) | — | 0.329 ⚠️ |

- 各层稀疏率差异大：浅层编码器 ~74–86%，深层解码器**高达 96–98%**
- 逐层阈值调优对平衡稀疏率与精度至关重要

## 4. 创新点分析 (≥3)
1. **免训练主干的动态稀疏化**：首次在冻结的预训练 DETR 上实现输入自适应的激活稀疏化，无需重训练
2. **分组门控 + Norm 监督**：用 12% 参数开销的门控网络，通过 L2 Norm 自动标注训练信号——优雅且低成本
3. **真实计算加速**：门控预测发生在第一层线性层**之前** → 实际跳过计算 → FLOPs 节省与稀疏率线性正比

## 5. 弱点 (≥5)
1. **仅验证 DETR**：未在 CNN 检测器（YOLO、Faster R-CNN）上验证——CNN 的激活稀疏模式可能与 Transformer MLP 不同
2. **全层同时应用性能下降**（mAP 0.399→0.329）：需要逐层阈值调优 → 超参搜索成本
3. **仅 MLP 层**：未涉及注意力层或卷积层的稀疏化
4. **代码未公开**
5. **COCO 仅验证集**：无 test-dev 结果，无小目标/中等/大目标的分尺度报告
6. **门控训练依赖 GT 激活 Norm**：若 baseline 模型本身对某些输入全激活（稀疏率低），门控将学到"始终通过"——无法强制稀疏
7. **MLSP Workshop**（非主会）：审稿标准低于 CVPR/ICCV/NeurIPS 主会

## 6. 对本项目的启发

### 🎯 对 #5 (语义熵 P2 稀疏化) 的**直接工程模板**
MGS 提供了 #5 最需要的**实现范式**：
- **门控位置**：在 P2 检测头的第一个卷积层之前插入（类比 MGS 在 MLP 第一个线性层之前）
- **门控结构**：单层 Conv1×1 + Sigmoid → 预测 P2 空间位置的分组激活 mask
- **监督信号**：用语义熵图的**局部统计量**（而非 MGS 的激活 Norm）作 GT：
  - 高熵区域 → label=1（需要计算）
  - 低熵区域 → label=0（可跳过）
- **训练方式**：冻结 YOLO11 backbone + Neck，仅训练门控卷积层 → 5–10 epochs 收敛
- **分组粒度**：P2 特征图的通道分组（如 256 通道 → 16 组 × 16 通道/组，用 16 个 gate）

### 对 #11 (高频能量 P2 稀疏化) 的启示
- 门控的**输入来源**可替换：不输入 P2 特征本身，而是输入 FFT 高频能量图（单通道）→ 门控更轻量
- MGS 的 "group-wise gating" 天然适合 #11：高频能量图按空间分块（如 8×8 patches），每块一个 gate

### ⚠️ 关键差异需注意
- MGS 是**通道级稀疏化**（跳过神经元组）→ #5/#11 需要**空间级稀疏化**（跳过某些空间位置的计算）
- 但 MGS 的**门控训练范式**（冻结主干 + BCE + Norm 监督）完全可迁移到空间门控

## 7. 可继续研究的问题
- MGS 的分组门控在 CNN 检测器（YOLO）的卷积层上是否同样有效？（CNN 激活稀疏模式 vs Transformer MLP）
- 空间级门控（每个 spatial location 一个 gate）vs 通道级门控（每组 channel 一个 gate）的精度-效率 trade-off
- 门控阈值是否可学习/自适应（而非固定 0.5）→ 对应 #5 的 "自适应熵阈值"（#8 变体）
