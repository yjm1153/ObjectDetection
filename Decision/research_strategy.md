> **2026-07-18 策略调整**：推翻 2026-07-17「YOLO + DETR 双轨并行」决策。DETR 不再作为独立研究轨道，降级为 YOLO 主线的**交叉融合灵感源**。Idea 分类见 innovation_ranking.md 🧭 区块;任务分区见 TASKS.md。

| 角色 | 定位 | 现有积累 | 近期任务 | 资源占比 |

**为什么从双轨回归 YOLO 为主**：①YOLO 积累最深（15个Idea + 4个设计文档），分散精力到 DETR 独立研究得不偿失；②DETR 的五大专属机制（Query/Cross-Attention/Bipartite Matching/Encoder-Decoder非对称/Token序列）在 YOLO CNN 架构中无直接对应，纯 DETR 创新无法迁移；③但 DETR 的**判据层**（频域先验/密度感知/遮挡建模）和**概念层**（条件计算/自适应分配）仍可为 YOLO 研究提供交叉灵感——这是 DETR 阅读的剩余价值。

**DETR 论文阅读新原则**：每篇 DETR 论文读后必问"这对 YOLO 有什么用？"——无法回答的，不做深读，不进 KB，不生成 Idea。

## 🔴 战略扩展：四维问题空间（2026-07-18）

> **背景**: 当前 26 个 Idea 过度集中在"条件计算/稀疏化→省算力"单一维度。VisDrone 真实场景中同等重要的**密集遮挡**、**旋转框**、**极端尺度变化**被系统性忽略。本节定义三个新方向的推进策略。

### 为什么必须扩展

| 当前状态 | 风险 | 扩展后收益 |
|----------|------|-----------|
| 26 个 Idea, 15 个围绕 P2 稀疏化 | 论文故事 = "我们找到了省算力的好办法"——单薄 | 论文故事 = "我们系统解决了无人机视角检测的四个核心难题"——完整 |
| 密集遮挡方向 0 Idea | 审稿人: "VisDrone 停车场场景的密集遮挡呢?"——致命问题 | Repulsion Loss / 密度感知 NMS / 遮挡建模——全新叙事线 |
| 旋转框方向仅 #17(低优先级) | 无人机视角天然适合 OBB，忽略 = 脱离应用场景 | OBB + 条件计算 = 无人做的交叉空白 |
| 尺度变化仅从 P2 稀疏化切入 | 只解决"算不算"，没解决"怎么更好地跨尺度建模" | 自适应感受野 / 尺度感知融合 / 极端小目标增强 |

### 扩展原则

1. **不与现有 Idea 冲突**：新方向 = 新维度，不替代不挤占条件计算线
2. **优先最薄弱的**：密集遮挡 > 旋转检测 > 尺度变化（尺度已有部分积累）
3. **遵循现有工作流**：每方向走完整 `LITERATURE → KNOWLEDGE → GAP → IDEA → DESIGN` 循环
4. **交叉优先**：探索"密集 × 旋转"、"尺度 × 遮挡"等交叉空白——这些通常 novelty 最高
5. **实验先行轻量验证**：优先找 CPU 可执行的统计分析型预实验（参考 #19 的模式）

### 🟪 DETR 交叉融合：从独立轨道到 YOLO 灵感源（2026-07-18 策略修订）

> **2026-07-18 策略转向**：推翻同日早些时候的「DETR 轨道强化」计划（DX1-DX5）。DETR 不再作为独立研究方向，而是 YOLO 主线的**交叉融合辅助**。

#### 为什么降级

| 理由 | 说明 |
|------|------|
| **机制不可迁移** | DETR 的 Query/Cross-Attention/Bipartite Matching 在 YOLO CNN 架构中无直接对应——纯 DETR 创新无法服务 YOLO 主线 |
| **精力分散** | YOLO 已有 15 个 Idea + 4 个设计文档 + 三维度扩展，分散精力到 DETR 独立研究会稀释 YOLO 深度 |
| **判据层可保留** | 频域先验/密度感知/遮挡建模等**判据层**是架构无关的——这些是 DETR 阅读的剩余价值 |
| **概念层可迁移** | 条件计算/自适应分配/合并>丢弃等**概念洞察**可启发 YOLO 设计——不需要 DETR 载体 |

#### DETR 阅读新协议

每篇 DETR 论文读后必须通过「**YOLO 迁移过滤器**」：

```
DETR 论文 → 提取核心洞察
         → 问：这个洞察的哪个部分与架构无关？
         → 问：能否用 CNN/YOLO 的机制重新实现？
         → ✅ 能 → 生成交叉 Idea（标记 ⬜ C类，YOLO 优先验证）
         → ❌ 不能 → 仅记录不深读，不进 KB，不生成 Idea
```

#### 保留的 DETR 交叉线

| 交叉线 | 保留理由 | YOLO 迁移路径 | 优先级 |
|--------|---------|-------------|--------|
| **判据层通用**（#30 频域判据/#35 频域遮挡先验/#36 熵-遮挡相关） | 判据本身架构无关——频域/熵/密度特征不依赖 DETR 或 YOLO | CNN P2 特征图替代 DETR token 序列作为判据输入 | 🟡 保留·低优先级 |
| **概念层迁移**（Dynamic DETR 合并式>丢弃式、RCDR 中心正则） | 工程洞察可跨架构 | MTA 聚合→P2 特征合并策略；RCDR→YOLO 回归头正则 | 🟡 保留·低优先级 |
| **密度×频域交叉**（#31 双维联合判据） | 双轨共享 Gap，首验选 YOLO | YOLO P2 特征图上的密度×频域联合门控 | 🟡 保留·低优先级 |
