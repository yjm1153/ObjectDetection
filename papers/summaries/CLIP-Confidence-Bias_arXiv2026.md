# Confidence Scores in Open-Vocabulary Detection Are a Biased Mixture of Scale and Semantics

## 1. 问题 (Problem)
CLIP-based 开放词汇检测器（GroundingDINO, OWL-ViT, YOLO-World）的置信度分数**不是可靠的定位概率估计**，而是系统性地包含两种偏差：
1. **尺度偏差 (Scale Bias)**：大目标置信度膨胀（α=+0.064, r=0.579, p=1.29×10⁻⁵⁸）
2. **语义偏差 (Semantic Bias)**：通用查询被抑制（β=-0.705, p=5.23×10⁻⁴¹）

这两种偏差是 CLIP 的图像级对比预训练**结构性导致的**，无法通过阈值调参消除。

## 2. 方法 (Method)

### 2.1 偏差诊断
- COCO + LVIS (1203 类) 双数据集验证
- GroundingDINO / OWL-ViT / YOLO-World 三检测器交叉验证
- 发现：**即使 oracle per-scale 阈值**也无法消除问题（小目标 ΔF1=+0.001 vs 大目标 ΔF1=+0.102 → 100× 差距）

### 2.2 参数自由温度缩放修正
- 推理时应用温度参数调整 Softmax
- 小目标 Recall@10 提升 **+19.6%** (p<0.01)
- 代价：pooled-ranking precision 下降 → **偏差只能部分纠正，无法免费逆转**

## 5. 对 Idea#5 的关键启示

### ⚠️ 核心警告
**CLIP 特征天然歧视小目标！** 这意味着：
1. Idea#5 直接使用 CLIP 语义熵作为 P2 门控信号 → **门控本身可能带偏差**
2. 小目标区域 CLIP 相似度被系统性压低 → 语义熵可能异常高 → 过度激活 P2 → 失去稀疏化意义
3. 或者反过来：小目标 CLIP 特征不可靠 → 语义熵不可信 → 门控失效

### 🛡️ 应对策略
1. **注意力熵替代语义熵**：借鉴 ViCrop-Det 的 SAE → 使用 decoder 注意力熵代替 CLIP 语义熵 → 但 YOLO 无 decoder cross-attention
2. **尺度校正语义熵**：在语义熵计算前对 CLIP 嵌入应用温度缩放 → 补偿尺度偏差
3. **双源门控**：语义熵 (CLIP) + 高频能量 (免 VLM) → #15 三源融合方案天然规避此问题
4. **免 VLM 路线优先级提升**：#11 高频能量引导 P2 稀疏化 不受 CLIP 偏差影响 → 风险更低
5. **查新价值**：CLIP 偏差 + 小目标检测的结合点 **未见于任何已查论文** → Idea#5 差异化叙事增强
