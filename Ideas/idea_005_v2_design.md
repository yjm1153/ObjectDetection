## 一、三篇论文交叉分析

### 1.1 三篇论文对 Idea#5 的不同角色

```
                    ┌─────────────────────────────────┐
                    │         Idea#5 v2.0              │
                    │  语义熵引导 P2 特征稀疏化         │
                    └─────────────────────────────────┘
                            ↑           ↑           ↑
                    ┌───────┘           │           └───────┐
                    │                   │                   │
           ViCrop-Det             CLIP-Bias             DM-EFS
        「范式验证者」          「风险揭示者」         「演化基线」
                    │                   │                   │
    ✅ 熵引导计算分配     ⚠️ CLIP特征歧视       ✅ P2浅层特征价值
       范式已验证              小目标                  获ICCV验证
    ✅ 可作正交佐证        ⚠️ 语义熵门控          ✅ 粗粒度baseline
       引用                     可能带偏差               演化路径清晰
    ✅ "零注意力盲区"      ✅ 温度缩放可           ✅ 尺寸引导→语义
       →VLM覆盖更全           部分修正                  引导提升链
```

### 1.2 交叉分析矩阵

| 维度 | ViCrop-Det | CLIP-Bias | DM-EFS | Idea#5 定位 |
|------|-----------|-----------|--------|------------|
| 引导信号 | 注意力熵（内生） | CLIP置信度（外生） | 目标尺寸（预测） | **语义熵（VLM外生）** |
| 信号可靠性 | 高（模型自身信号） | ⚠️ 结构性偏差 | 高（尺寸易预测） | 🟡 需校正（CLIP-Bias警示） |
| 架构 | DETR专属 | 通用（CLIP-based） | YOLOv7 | **YOLO11/YOLOE** |
| 作用阶段 | 推理时 | 推理时 | 训练+推理 | **训练+推理（模型内部）** |
| 粒度 | 图像裁剪区域 | 检测框级 | 图像级（3档） | **空间token级** |
| 计算范式 | 重检测（re-detect） | 分数校准 | 特征通道选择 | **特征空间稀疏化** |
| 小目标处理 | 裁剪放大→重检测 | ⚠️ 被系统性抑制 | 启用更多浅层特征 | **保留P2前景区计算** |

### 1.3 关键洞察：三条证据链

**证据链 1 — 熵范式有效性（ViCrop-Det → #5）**
- ViCrop-Det: SAE 注意力熵 → VisDrone +1.4, COCO APS +2.1
- 推论：注意力熵能定位小目标 → 语义熵也应该能（且覆盖"零注意力盲区"）
- 论文写作：ViCrop-Det 作为 Related Work 正交佐证引用

**证据链 2 — P2 浅层特征价值（DM-EFS → #5）**
- DM-EFS: P2 浅层特征 → VisDrone +2.11, 开销极小(38.24 vs 39.71 FPS)
- 推论：P2 特征对小目标至关重要，但 DM-EFS 用粗粒度尺寸引导→可用语义引导做更精细的选择
- 论文写作：DM-EFS 作为"粗粒度 baseline"，展示提升链

**证据链 3 — CLIP 偏差风险（CLIP-Bias → #5）**
- CLIP-Bias: 小目标 CLIP 置信度被系统性压低(r=0.579, p<10⁻⁵⁸)
- 推论：原始语义熵在小目标上可能不可靠 → 必须显式校正
- 论文写作：CLIP-Bias 作为 motivation 的实证支撑 → "现有 VLM 特征存在结构性尺度偏差，我们提出..."

## 二、统一叙事：从尺寸引导到语义引导的演化

### 2.1 演化链

```
阶段0: 均匀计算（Baseline YOLO11）
  P2 所有位置等权计算 → 算力浪费在背景
  └─ VisDrone AP: 基线水平

阶段1: P2 头 + 截短 backbone（Idea#6 SLE）
  添加 P2 头但截短 backbone 控制总计算量
  └─ 最简单的"加P2不加算力"方案
  └─ 缺陷：P2 仍是稠密计算，没有选择性

阶段2: 尺寸引导 P2 特征复用（DM-EFS 路线）
  根据图像级目标尺寸预测选择 P2 特征子集
  └─ 粒度：图像级（3档）
  └─ 缺陷：同一图像内不同区域的小/大目标用相同策略

阶段3: 语义引导 P2 token 稀疏化（Idea#5 v2.0 ⭐）
  逐 token 级语义熵门控 → 前景区精细计算、背景区跳过
  └─ 粒度：空间token级（连续值）
  └─ 优势：同一图像内自适应——小目标区域保留、大目标/背景区域跳过
```

### 2.2 核心 claim

> **P2 特征层的计算分配应该由"这里有什么"（语义）决定，而非"目标多大"（尺寸）决定。**
>
> 尺寸引导（DM-EFS）回答"是否需要浅层特征"，语义引导（#5）回答"哪些位置需要浅层特征"——后者更精细且可覆盖尺寸引导无法处理的场景（大小目标混合在同一图像）。

### 3.2 语义熵计算模块（核心创新）

```
输入：
  - F_P2: P2 特征图 (B, C_P2, H_P2, W_P2)
  - E_text: 类别文本嵌入 (K, D) — YOLOE 离线预计算
  - E_obj: YOLOE 物体嵌入提取器（冻结）

步骤：
  1. 物体嵌入提取
     E = YOLOE_ObjEmbed(F_P2)  # (B, H, W, D)
     或：从原始图像裁剪 P2 感受野区域提取

  2. 相似度计算
     S = E @ E_text.T            # (B, H, W, K)
     S_norm = softmax(S / τ_temp, dim=-1)  # τ_temp 温度缩放校正

  3. 语义熵计算
     H = -Σ_k(S_norm * log(S_norm + ε)) / log(K)
     # H ∈ [0, 1], 高→不确定(前景), 低→确定(背景)

  4. 温度缩放校正（CLIP-Bias 应对）
     τ_temp[i] = τ_base * (1 + α * scale_factor[i])
     # α ∈ [-0.2, 0.2] 可学习或固定
     # scale_factor 基于 P2 感受野的目标估计尺寸

  5. 门控图生成
     G = σ((H - τ_thresh) * β)  # sigmoid 软化
     # 或：G = H > τ_thresh（硬门控，训练时用 Gumbel-Softmax）

输出：
  - G: 门控图 (B, 1, H_P2, W_P2), G ∈ [0, 1]
  - H: 语义熵图 (可选，用于可视化/蒸馏)
```

### 3.3 P2 稀疏处理模块

```
方案 A — 掩码卷积（训练友好，推荐首发）
  F_P2_sparse = Conv_P2(F_P2) ⊙ G + F_P2 ⊙ (1 - G) * γ
  # 前景区: 正常卷积; 背景区: 残差连接（保留微弱信号）+ 可学习缩放 γ
  # FLOPs 节省: G 的稀疏度 × Conv_P2 FLOPs

方案 B — 稀疏卷积（推理高效，工程挑战）
  indices = where(G > τ)
  F_P2_foreground = Conv_P2(F_P2[indices])  # 只计算前景区
  F_P2_output = scatter(F_P2_foreground, indices)
  # FLOPs 节省: 实际节省 (1 - sparsity) × Conv_P2 FLOPs
  # 工程难点: batch 内稀疏模式不一致 → 需要 per-sample forward

方案 C — 混合（推荐终版）
  训练: 方案 A（掩码卷积 + Gumbel-Softmax 端到端训练）
  推理: 方案 B（转为稀疏卷积，torch.sparse 或 MIGX 加速）
```

### 3.4 CLIP-Bias 应对方案（三选一或组合）

| 策略 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **A. 温度缩放** | τ_temp 根据尺度调整 → 补偿 CLIP 尺度偏差 | CLIP-Bias 直接验证(+19.6% R@10) | 不能完全消除偏差 |
| **B. 免 VLM 双判据** | 语义熵(H_s) + 高频局部异常度(H_f) → OR 融合 | 高频判据不受 CLIP 偏差影响 | 两路信号融合增加复杂度 |
| **C. 语义熵归一化** | 在 P2 空间上做局部对比度归一化(LRN)，而非用原始熵值 | 自适应，无需尺度估计 | 可能过度归一化→前景/背景对比度降低 |

**推荐策略**: A（温度缩放，作为 baseline）+ B（双判据 OR 融合，作为增强版）。C 作为 ablation 备选。

### 3.5 损失函数

```
L_total = L_det + λ_gate * L_gate + λ_sparse * L_sparse

L_det: YOLO 标准检测损失(cls + box + dfl)
L_gate: 门控正则化
  - L_gate = BCE(G, G_pseudo)  # 可选：伪标签监督
  - 或 L_gate = λ_ent * mean(H⊙G)  # 鼓励高熵区域门控激活
L_sparse: 稀疏度正则化
  - L_sparse = |mean(G) - target_sparsity|  # 目标稀疏度约束
  - target_sparsity: 预设（如 0.7 → 70% P2 特征跳过）
```

### 3.6 误剪兜底机制（Token Cropr LLF 启发）

```
P2 被稀疏跳过的位置 ──→ P3 上采样填充 + 1×1 conv 轻量复活
  - 零/极少参数（~C_P2 × C_P3 × 1×1）
  - 不逆转稀疏化收益（只填充被误剪区域）
  - 在检测头前融合：F_P2_final = G⊙F_P2_processed + (1-G)⊙Upsample(F_P3)_processed
```

## 四、实验设计

### 4.1 预实验（CPU 可行，最高优先级 🔴）

**目标**: 一次实验定 Idea#5 生死

| 实验 | 方法 | 度量 | 通过标准 |
|------|------|------|----------|
| E0.1 | 在 VisDrone 验证集上用 YOLOE 提取语义熵图 | 可视化 | 熵图在前景区明显高于背景 |
| E0.2 | 计算 H @ GT 小目标区域 vs H @ 背景区域 | IoU/H 相关性 | 显著差异(p<0.05) |
| E0.3 | 确定最佳 τ_thresh（平衡召回率与稀疏度） | ROC 曲线 | AUC > 0.7 |
| E0.4 | CLIP-Bias 验证：小目标 vs 大目标的语义熵分布差异 | KS 检验 | 量化偏差量级→决定校正策略 |

### 4.2 消融实验（需要 GPU）

| 消融 | 组件 | 预期 |
|------|------|------|
| A0 | YOLO11n + P2 (SLE baseline, #6) | AP_s 基线 |
| A1 | A0 + 语义熵门控（硬阈值） | AP_s ↑, GFLOPs ↓ |
| A2 | A1 + 温度缩放校正 | AP_s ↑↑ (恢复偏差损失) |
| A3 | A2 + Gumbel-Softmax 软化 | 训练稳定性 ↑ |
| A4 | A3 + LLF 误剪兜底 | AP_s ↑, 召回率 ↑ |
| A5 | A4 + 双判据 OR 融合（+ #11 高频） | 鲁棒性 ↑, AP_s ↑↑ |

### 4.3 对照实验

| 对照 | 基线 | 目标 |
|------|------|------|
| C1 | DM-EFS 尺寸引导 | 证明语义引导 > 尺寸引导 |
| C2 | 随机门控 | 证明门控不是"随机稀疏化的正则化效果" |
| C3 | GT 门控（oracle） | 门控理论上限 |
| C4 | YOLOv12 (无 P2) | 证明 P2+稀疏化 > 纯注意力架构 |
