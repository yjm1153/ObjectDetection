# DQ-DETR: DETR with Dynamic Query for Tiny Object Detection

> ECCV 2024 | National Taiwan University | arXiv: 2404.03507

## 一句话总结
首个专为微小目标检测设计的 DETR 类模型：通过密度预测**动态调整 query 数量和位置**，AI-TOD-V2 30.2% AP（SOTA），VisDrone 37.0% AP。

---

## 核心机制

### 三大组件

| 组件 | 功能 | 实现 |
|------|------|------|
| **CCM** (Categorical Counting Module) | 预测图像中目标数量→四档分类 | 密度提取器 + 2层线性分类头 |
| **CGFE** (Counting-Guided Feature Enhancement) | 用密度图增强编码器特征 | 空间交叉注意力 + 通道注意力 |
| **DQS** (Dynamic Query Selection) | 自适应调整 query 数量和位置 | Top-K 分数选择 + 锚框偏移预测 |

### CCM — 分类优于回归

- 输入：编码器最大分辨率特征图 S₁
- 密度提取器：1×1 卷积（通道降维）+ 膨胀卷积序列（扩大感受野）
- 四档分类（AI-TOD-V2）：N≤10 / 10<N≤100 / 100<N≤500 / N>500
- **回归直接崩塌**：14.9 AP vs 分类 30.2 AP（目标数范围 1–2267 太宽）
- 分类准确率：94.6%（但 N>500 仅 56.6%，仅 46 个训练样本）

### DQS — Query 数量 + 位置动态化

| 密度档位 | Query 数 K |
|---------|-----------|
| N ≤ 10 | 300 |
| 10 < N ≤ 100 | 500 |
| 100 < N ≤ 500 | 900 |
| N > 500 | 1500 |

**位置增强**：CGFE 特征 → FFN 分类打分 → Top-K 选择 → 
- Content query: Linear(F_select)
- Position query: FFN 预测 Δb = (Δx, Δy, Δw, Δh) 加到原始锚框

## 与本项目的关系

### → Idea #5（P2 稀疏化）

| DQ-DETR 概念 | → P2 场景映射 |
|-------------|-------------|
| CCM 密度预测 | P2 目标密度预测 → 自适应稀疏率 |
| 四档密度 → 四档 query 数 | 四档密度 → 四档 P2 token 保留率 |
| DQS 动态数量分配 | 动态 P2 token 预算分配 |
| 分类优于回归 | P2 密度预测也应用分类而非回归 |

### 关键区别
- DQ-DETR 调整 DETR decoder query 数量
- Idea #5 调整 P2 检测头 token 稀疏率
- **但核心洞察一致**：目标密度 → 计算资源自适应分配

### 新启示
1. **密度自适应稀疏率**：不是所有图像用同一个 P2 稀疏率——简单场景多剪，密集场景少剪
2. **分类优于回归**：密度预测用分档分类（避免极端值不稳定）
3. **FP/FN 双重优化**：稀疏场景减 FP，密集场景减 FN
